# Excalidraw tool specifications — replication reference

Replication-grade behavioral spec for every Excalidraw tool, extracted from the Excalidraw source
at `~/Downloads/excalidraw-master` by a 22-agent forensic workflow (each claim cites `file:line`),
then compared against LayoutForge. **The single most important invariant, repeated for every linear
tool: a line's visual THICKNESS comes from `strokeWidth / zoom` (a 1/2/4 px bucket) — NEVER from the
bounding-box height.** Violating this is what made our divider balloon into a giant bar; it is now fixed.

LayoutForge is a *semantic UI layout* tool (see `CLAUDE.md`): freeform-sketch tools
(freedraw, arrow, diamond, ellipse, lasso, laser, rectangle-as-primitive) are **out of scope by design** —
our shapes are semantic (card/container/frame/nav/text/button/input/table/chart/list/tabs/modal/icon/divider).

## selection

**Purpose:** Box-select tool to marquee-select multiple elements or start drag/move operations on selection  
**Keybinding:** Activated by clicking empty canvas with selection tool active; Alt+click from selection starts lasso mode instead

- **Creation:** Click on empty canvas (no hit element) initiates selection. Drag distance must exceed DRAGGING_THRESHOLD (10px) to become marquee. At pointerdown: selection element created (type="selection", x/y at click origin, width/height=0). If drag distance <= 10px on release, treated as empty-area deselect.
- **Geometry / thickness:** Selection element stored as bounding box: x, y, width, height (absolute scene coords). During drag: width = distance(originX, currentX), height = distance(originY, currentY), position adjusted so top-left is always min coords. Rendered as hollow rectangle at element.x + scrollX, element.y + scrollY with 0.5px offset for crisp 1px stroke. Visual stroke thickness = 1px / zoom (NOT from element.strokeWidth — zoom-dependent). Fill: semi-transparent blue rgba(0,0,200,0.04).
- **Interactions:** Shift+click adds elements to selection (appends to selectedElementIds). Ctrl/Cmd+Alt starts lasso mode mid-drag (converts to lasso tool, preserves selection if Shift held). Shift+drag grows marquee while maintaining existing selections. Alt+drag in active selection duplicates elements. Empty-area drag with no Shift clears prior selection. Marquee selects elements based on boxSelectionMode ('contain' or 'intersect' — stored in appState). On pointerup, selection element is cleared (set to null), selections finalized.
- **After create:** Selection element persists in state.selectionElement during drag. On pointerup: selections are finalized into state.selectedElementIds, state.selectionElement cleared. If selection was result of marquee (boxSelection.hasOccurred), only elements fully contained/intersecting marquee are selected. Click-empty-area deselect clears selectedElementIds before any drag begins.
- **Replication notes:** CRITICAL: selection element stroke is ALWAYS 1px / zoom, never from strokeWidth property. Thickness adapts to zoom in real-time rendering. Offset of 0.5px ensures crisp 1px line on canvas (compensates for line width centering). Width/height calculated as absolute distance along each axis; if drag is leftward/upward, position shifts to keep top-left corner. Must track hasOccurred flag to distinguish marquee select from click-move. Shift key adds to selection (makeNextSelectedElementIds preserves prior IDs). Ctrl/Cmd+Alt mid-drag must switch tool AND start lasso with correct initial point. Empty-area deselect must fire BEFORE checking drag (clearSelection called in handleSelectionOnPointerDown ~line 8738). boxSelectionMode affects which elements pass getElementsWithinSelection (contain vs intersect). Selection element lifecycle: created onPointerDown, mutated on each Move (dragNewElement updates x/y/width/height), cleared onPointerUp. Visual rendering always applies zoom to offset and lineWidth at render time."

## lasso

**Purpose:** Free-form lasso selection tool that draws a closed path to select elements by intersection or enclosure  
**Keybinding:** E

- **Creation:** On pointerDown at non-element location: call lassoTrail.startPath(x, y) with shift key state. Initializes LassoTrail which creates a new LaserPointer instance and begins tracking the drawn path. Trail starts with a single point.
- **Geometry / thickness:** LassoTrail extends AnimatedTrail which uses LaserPointer library (external @excalidraw/laser-pointer 1.3.1). Visual stroke is generated from getStrokeOutline(size / zoom) where size is derived from sizeMapping function. Size = min(easeOut(length_decay), easeOut(time_decay)). Thickness = strokeWidth / zoom (NOT bbox-derived). Path rendered as SVG with fill="rgba(105,101,219,0.05)" and stroke="rgba(105,101,219)". Animation is dashed (stroke-dasharray="7 7", stroke-dashoffset cycles 0 to -14 over 0.3s).
- **Interactions:** Shift key: when pressed at startPath or addPointToPath, sets keepPreviousSelection=true to append to existing selection instead of replacing it. Ctrl+Alt+click: forces re-lasso selection of already-selected elements. Alt key: on desktop during fromSelection mode, reverts to normal selection tool. Mobile: if lasso tool hits element directly, selects it immediately like normal selection.
- **After create:** On pointerUp: call lassoTrail.endPath() which closes the LaserPointer and moves it to pastTrails array. Separately, getLassoSelectedElementIds() is called with the finalized path to compute element hits. Hits are determined by: (1) enclosure test - polygon-includes-point with non-zero rule on element line segments, or (2) intersection test - lasso segments intersect element bounds. Text elements upgrade to container. Frame children are deselected if frame selected. Groups are auto-selected if any member selected. Desktop only: drag is blocked until next pointerDown after lasso selection unless Shift was held.
- **Replication notes:** CRITICAL: Thickness is NOT derived from stroke outline bbox. It comes from LaserPointer.getStrokeOutline(size/zoom) where size is computed from sizeMapping with time and length decay curves. Stroke is always divided by zoom when rendering (line 206 in animatedTrail.ts). Path simplification happens at 5/zoom distance (lasso/index.ts:204). Enclosure test uses non-zero winding rule - critical for self-intersecting paths. Text element containers are auto-selected, not text itself. Frames and their children never selected together - children deselected if frame hit. Groups auto-select all members. On desktop, blockDragging flag prevents immediate drag after lasso selection unless Shift was held at pointerDown. Mobile bypasses blockDragging if element was hit. Shift key modifies keepPreviousSelection before path starts AND at each addPoint call (app.tsx:10355).

- **Excalidraw source:** `packages/excalidraw/lasso/index.ts:40-69`, `packages/excalidraw/lasso/index.ts:71-88`, `packages/excalidraw/lasso/index.ts:164-170`, `packages/excalidraw/lasso/index.ts:172-209`, `packages/excalidraw/lasso/utils.ts:22-90`, `packages/excalidraw/lasso/utils.ts:92-108`, `packages/excalidraw/lasso/utils.ts:110-143`, `packages/excalidraw/animatedTrail.ts:40-59`

## rectangle

**Purpose:** Drag-create a closed rectangular shape with configurable stroke/fill at any size; click creates a zero-size invisible element (deleted on finalization)  
**Keybinding:** none

- **Creation:** Pointer down on canvas with activeTool.type="rectangle" → createGenericElementOnPointerDown() instantiates newElement() at origin (gridded unless Ctrl held), width=0, height=0. Element inserted into scene and set as state.newElement. Subsequent pointer moves trigger dragNewElement() which updates width/height as absolute distance from originX and originY, with negative values flipped to maintain correct x/y and positive dimensions. Click only (no drag beyond DRAGGING_THRESHOLD, width/height remain ~0) results in invisibly small element; on finalize, isInvisiblySmallElement() returns true (width===0 && height===0) and element is deleted.
- **Geometry / thickness:** Element stores absolute scene coords (x, y) as top-left corner; width and height as positive canvas units (scene space, zoom-invariant). Visual stroke thickness derived NOT from bbox but from element.strokeWidth (integer, default 2) at render time: context.lineWidth is set to element.strokeWidth when rendering via rough.js. However, selection box renders with context.lineWidth = 1 / appState.zoom.value (always 1px at viewport scale regardless of zoom). Rectangle geometry is axis-aligned; no implicit rotation.
- **Interactions:** Shift+drag: maintains 1:1 aspect ratio (getPerfectElementSize applied). Alt/Cmd+drag: resizes from center (shouldResizeFromCenter, doubles width/height then recenters). Ctrl: disables grid snapping during drag. Shift+Alt together: both aspect ratio and center resize. During drag, grid snapping applies via snapNewElement(). Dragging from bottom-right to top-left or right-to-left produces negative distances internally; dragNewElement() detects sign and repositions x/y accordingly, keeping width/height positive. Selection box around element drawn at renderSelectionBox() with strokeColor from renderConfig.selectionColor and lineWidth = 1/zoom.
- **After create:** On pointer up, element remains in scene as selected (state.selectedElementIds updated). If locked=false (default) and activeTool.locked=false, tool reverts to selection. If invisibly small (width===0 && height===0), finalize action deletes element. Element can be moved, resized, or have text added via double-click.
- **Replication notes:** CRITICAL: Stroke thickness is element.strokeWidth constant piped directly to rough.js via generateRoughOptions, NOT derived from visual bbox. Selection outline is always 1px at viewport (lineWidth = 1/zoom). Zero-width/height after finalize means isInvisiblySmallElement returns true, triggering deletion in actionFinalize. Grid snapping enabled by default unless Ctrl held (getGridPoint called in dragNewElement at line 12384-12388 of App.tsx). Aspect ratio lock (Shift) uses getPerfectElementSize which for non-linear types sets height = absWidth * sign(height) (line 151). Center resize (Alt) doubles both dimensions then recenters using originX/Y minus half the new size (lines 300-303). Selection updates happen in insertNewElement, which also assigns to state.newElement. All rendering uses context.translate to scroll offset before drawing at 0,0 with element's width/height.

## diamond

**Purpose:** Diamond shape tool — creates a rotated square (rhombus) with vertices at cardinal directions, fully parameterized by bounding-box width/height and rendered with stroke/fill styling  
**Keybinding:** none

- **Creation:** On pointerdown with activeTool.type="diamond": newElement({type:"diamond", x, y, strokeColor, backgroundColor, fillStyle, strokeWidth, strokeStyle, roughness, opacity, roundness, locked}). Width/height default to 0. During drag: dragNewElement() updates width/height via mouse delta, respecting shift-key aspect-ratio lock. Resize-from-center doubles width/height. Final dimensions stored when pointerup (or min threshold crossed).
- **Geometry / thickness:** Element stores x, y (top-left corner of bounding box), width, height. Diamond shape is defined by four points: topX=floor(width/2)+1, topY=0; rightX=width, rightY=floor(height/2)+1; bottomX=topX, bottomY=height; leftX=0, leftY=rightY. Visual thickness is NOT derived from bbox—it is element.strokeWidth (constant pixels in scene coords), rendered via roughjs which interprets strokeWidth in its options dict; at zoom, thickness scales as 1:1 (roughjs applies zoom internally). Roundness optionally applies cubic bezier corner radii via getCornerRadius(min(width,height), element).
- **Interactions:** Shift: maintain 1:1 aspect ratio during drag. Alt: resize from center. Ctrl+drag: disable grid snap. Double-click on diamond: enter text-edit mode (if container supports it). Cmd+drag: toggle binding preview.
- **After create:** Element becomes selected (selectedElementIds updated). If not text-edit mode, activeTool remains locked=false unless explicitly locked by user. Multi-element becomes null. newElement state is set; on pointerup cleared. Shape redrawn on next frame via ShapeCache.generateElementShape().
- **Replication notes:** 
CRITICAL GEOMETRY DETAILS:
- getDiamondPoints() must add +1 to topX, rightY, and bottomX to avoid rough.js errors on zero-size paths.
- Four points form a diamond: topX at center-x (0.5*width), topY at 0; rightX at full width; bottomX mirrors topX; leftX at 0. Always 4 vertices, never more.
- Width/height are SIGNED during drag (negative means drag direction reversed); must be flipped to positive before mutation.
- Roundness applies cubic bezier curves to each corner, NOT quadratic. Control points computed via getCornerRadius(min(width,height), element).

STROKE RENDERING:
- strokeWidth property in element is in scene-space pixels; roughjs receives this directly in options.strokeWidth.
- Zoom does NOT scale strokeWidth in roughjs options—roughjs applies zoom to the rendered path internally.
- Line join/cap set to "round" in drawElementOnCanvas before rc.draw().

CREATION FLOW:
- createGenericElementOnPointerDown() called when activeTool.type!="selection"/"lasso"/"text"/"arrow"/"line"/"freedraw"/"frame"/"eraser"/"hand"/"image".
- newElement() creates element with width=0, height=0; shape not visible until dragNewElement() updates dimensions.
- dragNewElement() is NOT called on pointerdown; it's called in onPointerMove handlers triggered by subsequent movement.
- Must respect shouldMaintainAspectRatio (shift-key), shouldResizeFromCenter (alt-key), and grid snapping.

SHAPE CACHING:
- ShapeCache.generateElementShape() uses roughjs RoughGenerator to create Drawable shape objects.
- Diamond with roundness=null returns generator.polygon([[topX,topY],[rightX,rightY],[bottomX,bottomY],[leftX,leftY]], options).
- Diamond with roundness uses generator.path() with cubic bezier SVG commands.
- Cache is invalidated on element mutation via ShapeCache.delete().

TESTING EDGE CASES:
- Minimum 1px width/height (dragNewElement checks width !== 0 && height !== 0 before update).
- Negative dimensions from drag reversal must be caught and flipped (see dragNewElement lines 296-304).
- Roundness="sharp" or null both valid; "round" not used for diamonds (uses element.roundness property instead).
- Empty points array on creation (no diamond-specific point data unlike arrow/line/freedraw).

## ellipse

**Purpose:** Create an axis-aligned ellipse by dragging from origin to opposite corner; stroked/filled shape with roughjs rendering.  
**Keybinding:** Click toolbar or activate via keyboard; no dedicated keybinding, selected from shape menu.

- **Creation:** Click and drag: pointerDown at origin (x0, y0), pointerMove/pointerUp at end (x1, y1). Element created on first pointerDown with width=0, height=0. dragNewElement resizes during drag. Element only persists if width ≠ 0 AND height ≠ 0 at pointerUp.
- **Geometry / thickness:** Stores element.x, element.y (top-left corner of bounding box), element.width, element.height. Visual size = bbox dimensions. Stroke thickness is NOT added to bbox; element.strokeWidth is a RoughJS rendering property (passed to RoughJS generator.ellipse). RoughJS renders ellipse centered at (width/2, height/2) with semi-major axes (width, height). Stroke is drawn OUTWARD from the mathematical ellipse shape, NOT included in bbox calculation. strokeWidth value affects only visual weight, not element.width/height.
- **Interactions:** Shift key: maintains aspect ratio (cursor heading vs origin determines if width or height controls size). Alt key: resizes from center (width/height doubled, position moves to origin - width/2, origin - height/2). Drag direction: if (x1 &lt; x0), element repositioned so origin is at x0 and width is absolute distance; same for y. Grid snapping applied when Ctrl/Cmd not held.
- **After create:** Element selected (added to selectedElementIds). newElement/selectionElement state cleared. Tool remains active (does not revert to selection).
- **Replication notes:** CRITICAL GEOMETRY: strokeWidth is NOT part of bounding box. The element stores integer strokeWidth (default 2), which RoughJS uses to control visual stroke thickness via its internal strokeWidth rendering option. When rendering in canvas, rc.draw(roughShape) applies this strokeWidth; the shape's perimeter expands outward but bbox (x, y, width, height) remains unchanged. This differs from SVG where stroke can expand bbox. Must call dragNewElement with width=distance(originX, gridX) and height=distance(originY, gridY) — NOT abs values, distance() handles sign. If width or height is 0, element is NOT mutated into scene (line 331: "if (width !== 0 && height !== 0)"). Aspect ratio lock uses Math.abs(y-originY) vs Math.abs(x-originX) to determine which dimension drives resize. Center resize: newX = originX - width/2, newY = originY - height/2 (after width/height are doubled). Element created immediately on pointerDown but only finalized with real dimensions during drag via scene.mutateElement().

## arrow

**Purpose:** Draw multi-point arrows with optional arrowheads, binding to shapes, and elbow/smooth curve modes  
**Keybinding:** Arrow is selected via toolbar or by choosing arrow tool in activeTool state; Alt+click adds additional points during creation

- **Creation:** Click on canvas to set start point (x, y), automatically initializes a 2-point arrow with points [[0,0], [0,0]] in local coords; subsequent clicks before pointerUp add points via Alt+click or automatic chaining; drag-and-release commits final arrow. Arrows start with endArrowhead="arrow" by default, startArrowhead=null. Grid snapping applied unless Cmd/Ctrl held.
- **Geometry / thickness:** Arrow is a linear element: stores x/y (top-left origin) + points array (LocalPoint[] = [x,y][] normalized to start at [0,0]). Visual thickness derived from strokeWidth property (number, pixels) passed to rough.js shape generation. CRITICAL: stroke visual thickness = strokeWidth / zoom.value (NOT bbox-derived). Width/height properties exist but are computed post-hoc from bounds of points. No width/height used for visual rendering of line itself.
- **Interactions:** Alt+click while in editing mode adds a new point at cursor; Shift+click selects/deselects multiple points; click within ~8px (LINE_CONFIRM_THRESHOLD) of last committed point finalizes; Ctrl disables binding; Alt at start binds arrow start inside element. Segment midpoints of elbow arrows can be dragged. Angle-locking via Shift constrains to 45-degree increments. Binding to shapes stores startBinding/endBinding with bindingElement ID and focus point.
- **After create:** Arrow enters multiElement mode for continued point addition; clicking start point or close to last committed point finalizes. If tool is locked, stays in arrow mode; otherwise reverts to selection tool. Finalized arrows with <2 points or invisible size are deleted. Arrows can bind to rectangles/diamonds/ellipses (bindingElement types); binding updates on drag.
- **Replication notes:** 1. GEOMETRY IS CRITICAL: rough.js strokeWidth is NOT zoom-scaled by the tool—zoom scaling happens in canvas context.scale() or as a visual multiplier ONLY for UI overlays. The arrow's visual thickness comes purely from element.strokeWidth passed to rough.js, which renders at zoom=1 intrinsically. Do NOT multiply strokeWidth by zoom in the shape generation. 2. Multi-point: each Alt+click adds points; clicks near (within 8px at current zoom) the last committed point OR the start point finalize the arrow. 3. ## snapping

**Purpose:** Magnetic object snapping, alignment guides, and gap/distribute snap to help align and space elements with pixel-perfect precision  
**Keybinding:** Ctrl/Cmd toggles snapping on/off when objectsSnapModeEnabled=false; always active when objectsSnapModeEnabled=true (unless holding Ctrl/Cmd)

- **Creation:** Snapping is not a creation tool; it's a post-creation behavior that affects element positioning during drag/resize. Three snap entry points: snapDraggedElements (dragging selected elements), snapResizingElements (resize handles), snapNewElement (new element being drawn). Gap snaps cache computed before interaction starts (maybeCacheVisibleGaps, maybeCacheReferenceSnapPoints called on pointerDown).
- **Geometry / thickness:** Snap targets are element corner points and centers (snapping.ts:198-313, getElementsCorners). For single elements: 4 corners + center (5 points). For non-rectangular: diamond/ellipse use mid-points (left, top, right, bottom) rotated by element.angle + center. Multi-element selection: bounding box corners + center. Gap snaps store pair-wise bounds [minX, minY, maxX, maxY] (snapping.ts:65-81, Gap type). Visual thickness: snap lines drawn at SNAP_WIDTH=1px constant, scaled by zoom: lineWidth = (1px / zoomValue) (renderSnaps.ts:10,30-31). Cross size = 2px scaled by zoom: size = (2px / zoomValue) (renderSnaps.ts:11,98-99). Gap snap lines draw with horizontal/vertical segments + cross markers at endpoints.
- **Interactions:** Snapping uses two-phase detect-apply cycle: (1) compute snap distance threshold = 8px / zoomValue (snapping.ts:41,48-50, getSnapDistance); (2) find nearest snaps within threshold across all axis-aligned reference points (point snaps) and gaps (gap snaps); (3) apply snapOffset to drag/resize. Point snap: compare every selected element snap point to every visible non-selected element snap point, measure offsetX/offsetY, keep snaps within threshold (snapping.ts:655-672, getPointSnaps). Gap snap: only process gaps with bounding-box overlap on the orthogonal axis (snapping.ts:476,545); three gap variants: center (when gap > selection size), side_left/side_right (horizontal), side_top/side_bottom (vertical). Modifier: Ctrl+drag when snapping disabled toggles ON; Ctrl+drag when snapping enabled toggles OFF (isSnappingEnabled, KEYS.CTRL_OR_CMD logic snapping.ts:178-183). Lasso tool: snaps only during drag of selected, not selection phase (snapping.ts:174-176). Arrow: never snaps (snapping.ts:188-189).
- **After create:** Snap is not creation-triggered; it's continuous during drag. Multi-axis: snaps are applied independently on X and Y (snapOffset.x and snapOffset.y computed separately, snapping.ts:751-753). Applied via dragSelectedElements(..., snapOffset) and in resize transforms (resizeX += snapOffset.x). Snap lines persist in appState.snapLines until next drag event or pointerUp. SnapCache destroyed on interaction end or disabled.
- **Replication notes:** CRITICAL DETAILS: (1) Snap distance = 8px at zoom=1, scales inversely with zoom (8/zoomValue). At zoom=2, threshold = 4px in canvas coords. (2) Line thickness = 1px / zoom, never bbox-derived. (3) Gaps computed ONCE before drag starts, cached in SnapCache to avoid recomputation. (4) Point snaps: compare EVERY selected snap point to EVERY reference snap point, O(m*n) in counts. (5) Gap snaps: only if bounding-box overlaps on perpendicular axis (minY..maxY must overlap gap.overlap for horizontal gap). (6) Multiple snaps at same distance kept (not cleared until closer snap found). (7) Snap offset is raw difference, NOT clamped; it can be negative. (8) Diamond/ellipse snap points are rotated to canvas space, not axis-aligned to bbox. (9) Gap snap line rendering draws WITH visual 'ticker' marks at gap midpoint + endpoints (8px / zoom spacing). (10) Zen mode disables line drawing, keeps only cross markers. (11) Rounding: 6 decimal places (Math.round(x * 1e6) / 1e6) for float precision. (12) Snap point deduplication by key (rounded x or y), not by distance. (13) Grid mode + Ctrl = no snap (snapping disabled if both active). (14) Single arrow element never snaps. (15) Container-bound elements excluded from gap computation. (16) Two-phase apply: (a) compute snap with dragOffset, (b) re-compute snap lines with dragOffset + snapOffset to render without offset jump.
