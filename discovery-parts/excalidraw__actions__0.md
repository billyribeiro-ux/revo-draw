## Cluster: excalidraw__actions__0

This cluster covers the first slice of Excalidraw's "action" registry — declarative command objects each created via `register(...)`. Every action object exposes a subset of: `name`, `label`, `icon`, `trackEvent`, `keywords`, `viewMode`, `predicate`, `keyTest`, `perform`, and an optional `PanelComponent` (a React render-prop). The `perform` function is the heart of each action; it takes `(elements, appState, formValue, app)` and returns either `false` (no-op) or an object describing the next `elements`/`appState` plus a `captureUpdate` directive (`IMMEDIATELY` = snapshot into history, `EVENTUALLY` = ephemeral/debounced, `NEVER` = no history capture). Mutations of elements go through `app.scene.mutateElement(...)`. This is the key contract a Svelte/Canvas reimplementation must mirror: actions are pure-ish reducers over `(elements, appState)` with a side-channel into the scene for in-place element mutation, and a history-capture classification.

### packages/excalidraw/actions/actionAddToLibrary.ts

Purpose: defines the single action that adds the current selection to the user's library as a new unpublished library item.

- `actionAddToLibrary` (registered action, `name: "addToLibrary"`, `label: "labels.addToLibrary"`, `trackEvent: { category: "element" }`) — L10-L65.
  - `perform: (elements, appState, _, app) => ...` — L13-L63. Collects the selected elements via `app.scene.getSelectedElements({ selectedElementIds, includeBoundTextElement: true, includeElementsInFrames: true })` (L14-L18). Guards against disallowed types: iterates `LIBRARY_DISABLED_TYPES`, and if any selected element matches, returns early with `captureUpdate: EVENTUALLY` and sets `appState.errorMessage` to `t(\`errors.libraryElementTypeError.${type}\`)` (L20-L30). Otherwise returns a Promise chain: `app.library.getLatestLibrary()` → prepends a new library item `{ id: randomId(), status: "unpublished", elements: selectedElements.map(deepCopyElement), created: Date.now() }` to the existing items via `app.library.setLibrary([...])` (L32-L44), then resolves to an appState carrying a success toast `t("toast.addedToLibrary")` (L45-L53). On `.catch`, resolves to an appState with `errorMessage: error.message` (L54-L62).
  - Notable: elements are **deep-copied** (`deepCopyElement`) so the library item is decoupled from the live scene. The action is async (returns a Promise) and never returns new `elements`/captures history into the document — all branches use `CaptureUpdateAction.EVENTUALLY`. Side effects: mutates the library store, not the scene. Invariant: `created` timestamp and `randomId()` make output non-deterministic (intentional, library-side).

### packages/excalidraw/actions/actionBoundText.tsx

Purpose: defines actions to bind/unbind text to a container and to wrap loose text inside a new rectangle container, plus two z-order helpers that keep the text element directly above its container.

- `actionUnbindText` (`name: "unbindText"`) — L54-L107.
  - `predicate`: true if any selected element `hasBoundTextElement` (L58-L62).
  - `perform`: for each selected element, gets its bound text element; re-measures the text at its natural size via `measureText(originalText, getFontString(boundText), lineHeight)` (L69-L73); reads the cached original container height via `getOriginalContainerHeightFromCache(element.id)` then `resetOriginalContainerCache(element.id)` (L74-L77); computes the freed text position via `computeBoundTextPosition(...)` (L78-L82); mutates the text element to detach it (`containerId: null`, new width/height/x/y, `text: originalText`) (L83-L90); mutates the container to drop the text from `boundElements` and restore its original height if cached (L91-L98). Returns `captureUpdate: IMMEDIATELY`.
  - Invariant: relies on the original-container-height cache so the container shrinks back to its pre-text-bound height; if no cache entry, keeps current height.
- `actionBindText` (`name: "bindText"`) — L109-L181.
  - `predicate`: true only when exactly 2 elements are selected, one is a text element and the other is a text-bindable container that does not already have bound text (L113-L139).
  - `perform`: disambiguates which selected element is the text vs the container (L143-L155); mutates the text element to bind it (`containerId`, `verticalAlign: MIDDLE`, `textAlign: CENTER`, `autoResize: true`, and `angle` copied from container unless it's an arrow → angle 0) (L156-L162); mutates the container's `boundElements` to add `{ type: "text", id }` (L163-L168); saves the container's current height into the original-container-height cache via `updateOriginalContainerCache` so unbind can restore it (L169-L173); calls `redrawTextBoundingBox` to relayout. Returns elements reordered by `pushTextAboveContainer(...)`, sets selection to just the container, `captureUpdate: IMMEDIATELY`.
- `pushTextAboveContainer(elements, container, textElement)` (internal) — L183-L201. Removes the text element from the array, finds the container index, and splices the text in at `containerIndex + 1` (i.e., immediately after/above the container in z-order), then calls `syncMovedIndices(updatedElements, arrayToMap([container, textElement]))` to fix the fractional indices. Returns the reordered array. Performance/ordering note: fractional indices (`syncMovedIndices`) keep z-order stable without renumbering everything.
- `pushContainerBelowText(elements, container, textElement)` (internal) — L203-L221. Inverse: removes the container, finds the text index, and splices the container in at `textElementIndex` (just below the text), then `syncMovedIndices`. Used by the wrap action where the container is created after the text already exists.
- `actionWrapTextInContainer` (`name: "wrapTextInContainer"`, label `labels.createContainerFromText`) — L223-L339.
  - `predicate`: at least one selected element is a text element not already bound to a container (L227-L233).
  - `perform`: for each unbound text element, creates a new `rectangle` container via `newElement(...)` inheriting current item style props (background/fill/stroke/roughness/width/style), with roundness derived from `appState.currentItemRoundness === "round"` choosing `ROUNDNESS.ADAPTIVE_RADIUS` vs `PROPORTIONAL_RADIUS` based on `isUsingAdaptiveRadius("rectangle")` (L254-L261); positions the container at `textElement.x/y - BOUND_TEXT_PADDING` and sizes it via `computeContainerDimensionForBoundText(text dimension, "rectangle")` (L264-L273); inherits the text element's `groupIds` and `frameId`. Then it rebinds any arrow bindings that pointed at the text element so they now point at the new container (L279-L308): it scans the text's `boundElements` for arrows, finds those linear elements, and rewrites their `startBinding`/`endBinding` `elementId` to the container id. Mutates the text element to bind it (`containerId`, `verticalAlign: MIDDLE`, `boundElements: null`, `textAlign: CENTER`, `autoResize: true`) (L310-L316), runs `redrawTextBoundingBox`, reorders via `pushContainerBelowText`, and records the new container id in `containerIds`. Returns the updated elements, sets selection to the set of newly created containers, `captureUpdate: IMMEDIATELY`.
  - Geometry note: `BOUND_TEXT_PADDING` is the per-side inset between container edge and text; container dimensions are not raw text+2*padding but go through `computeContainerDimensionForBoundText` which accounts for shape-specific padding (rectangle vs ellipse vs diamond).

### packages/excalidraw/actions/actionDeleteSelected.tsx

Purpose: the delete action plus its supporting helpers — handles deleting selected elements, deleting frames (orphaning vs. cascading their children), deleting individual linear-element points, and reselecting the next eligible element in an editing group.

- `deleteSelectedElements(elements, appState, app: AppClassProperties)` (internal) — L39-L187. The core deletion reducer.
  - Builds `framesToBeDeleted`: the set of selected frame-like element ids (L44-L49).
  - For each frame to delete, walks its `getFrameChildren` and marks each child for selection-after-delete (mapping bound text back to its container) so the children survive but become selected (L57-L74).
  - Maps over all elements (L78-L128): if an element is selected, it may (a) become selected-and-orphaned if it belongs to a deleted frame, (b) be skipped if its container belongs to a deleted frame, (c) have its bound elbow-arrow bindings nulled (mutating the arrow's `startBinding`/`endBinding` in place when this element was the binding target), then be marked `isDeleted: true` via `newElementWith`. Elements that merely belong to a deleted frame (but aren't themselves selected) get `frameId: null` (orphaned, not deleted) and are selected. Bound text whose container is selected is also deleted (L124-L126).
  - Then (L130-L170) computes `nextEditingGroupId`: if still editing a group, selects the next eligible non-deleted element in that group, or walks up to the super-group if the group is now too small.
  - Returns `{ elements: nextElements, appState: { ...appState, ...selectGroupsForSelectedElements({selectedElementIds, editingGroupId}, nextElements, appState, null) } }`. Invariant: deleting a frame deletes the frame but **orphans** (does not delete) non-selected children; selected children of a deleted frame are also deleted.
- `handleGroupEditingState(appState, elements): AppState` (internal) — L189-L206. If editing a group, finds remaining (non-deleted) sibling elements in that group and, if any remain, sets selection to the first sibling; otherwise returns appState unchanged. Keeps a valid selection after a deletion inside a group edit.
- `actionDeleteSelected` (`name: "deleteSelectedElements"`, label `labels.delete`, icon `TrashIcon`, `trackEvent: { category: "element", action: "delete" }`) — L208-L329.
  - `perform` — L213-L304. Branches:
    1. **Linear-element point editing** (`appState.selectedLinearElement?.isEditing`): looks up the element; returns `false` if not found or if no points selected (deliberate no-op to avoid accidentally deleting the whole element). If all points are selected, marks the whole linear element `isDeleted: true` and clears `selectedLinearElement`. Otherwise calls `LinearElementEditor.deletePoints(linearElement, app, selectedPointsIndices)` and adjusts the selected point index to the previous point (or 0). All branches capture `IMMEDIATELY`. (L214-L273)
    2. **General deletion**: calls `deleteSelectedElements`, then `fixBindingsAfterDeletion(nextElements, deletedElements)` to repair arrow bindings, then `handleGroupEditingState`, and returns appState resetting the active tool to the preferred selection tool, clearing `multiElement`/`newElement`/`activeEmbeddable`/`selectedLinearElement`. `captureUpdate` is `IMMEDIATELY` if something was actually selected, else `EVENTUALLY` (L275-L303).
  - `keyTest`: `BACKSPACE` or `DELETE` without CtrlOrCmd (L305-L307).
  - `PanelComponent` — L308-L328. Reads `useStylesPanelMode()` hook; renders a `ToolButton` (TrashIcon) disabled when nothing is selected, applying a mobile background style (`MOBILE_ACTION_BUTTON_BG`) when in mobile mode and the compact-other-properties popup isn't open. State source: the `useStylesPanelMode` React hook from `../components/App`.
  - Side effects/invariants: arrow bindings are repaired post-deletion; elbow-arrow bindings are nulled in-place during the map; deletion is non-destructive to the array (uses `newElementWith` to flag `isDeleted` rather than removing entries, preserving fractional indices and history reversibility).
