## Cluster: element__src__4

This cluster covers the geometry-on-grid layout helper, the static/interactive element rasterizer, the resize/rotate transform engine, the resize hit-test, the reactive `Scene` document model, box-selection logic, and the roughjs shape generator + cache. All math is world-space (scene) unless explicitly noted; the renderer converts to device pixels via `window.devicePixelRatio`.

### packages/element/src/positionElementsOnGrid.ts

Lays out a flat list (or list of atomic groups) of elements into a roughly-square grid centered on a point — used after AI/flowchart generation. (Author comment: "TODO rewrite (mostly vibe-coded)".)

- **`positionElementsOnGrid<TElement extends ExcalidrawElement>(elements: TElement[] | TElement[][], centerX: number, centerY: number, padding = 50): TElement[]`** — L7-L112. Normalizes input: if `elements[0]` is an array, treats each sub-array as an atomic unit; otherwise wraps each element into its own singleton unit (L21-L23). Column count = `Math.max(1, Math.ceil(Math.sqrt(numUnits)))` (L29) → near-square grid. Slices units into rows of `numColumns` (L33-L35). For each row computes per-unit bounds via `getCommonBounds(unit)`, row width (sum of unit widths + inter-unit padding, no trailing pad), and max unit height (L40-L71). Total grid height = sum of row max-heights + `(rows.length-1)*padding` (L74-L75). Starting `currentY = centerY - totalGridHeightWithPadding/2` to center vertically (L78); each row's `currentX = centerX - rowWidth/2` to center horizontally (L85). For each unit, offset = (currentX - originalMinX, currentY - originalMinY), applied to every element via `newElementWith` (immutable copy) translating only `x`/`y` (L87-L105). Advances `currentX` by unit width + padding; `currentY` by row max-height + padding (L104-L109). Returns NEW elements (does not mutate). Invariant: relative positions within an atomic unit are preserved (single shared offset). Side effects: none (pure, allocates copies).

### packages/element/src/renderElement.ts

Rasterizes a single element to an offscreen per-element canvas (cached) then blits it to the target 2D context with rotation/scale/crop, plus opacity computation and freedraw outline segmentation; this is the core of the static-scene render pipeline.

- **`isPendingImageElement(element, renderConfig): boolean`** — L85-L90. True if element is an initialized image whose `fileId` is not yet in `renderConfig.imageCache`.
- **`getCanvasPadding(element): number`** — L92-L106. Per-type offscreen-canvas padding so strokes/arrowheads aren't clipped: freedraw → `strokeWidth*12`, text → `fontSize/2`, arrow → 40 (if any arrowhead) else 20, default 20. Note L99 bug: `element.endArrowhead || element.endArrowhead` (should likely be start||end).
- **`getRenderOpacity(element, containingFrame, elementsPendingErasure, pendingNodes, globalAlpha = 1): number`** — L108-L132 (exported). Combines frame opacity × element opacity / 10000 × globalAlpha (both stored 0-100 → product /10000 normalizes to 0-1). If the element, a pending flowchart node match, or its containing frame is pending-erasure, multiplies again by `ELEMENT_READY_TO_ERASE_OPACITY/100`. Returns 0-1 alpha.
- **interface `ExcalidrawElementWithCanvas`** — L134-L147. Cache record: `{ element, canvas, theme, scale, angle, zoomValue, canvasOffsetX, canvasOffsetY, boundTextElementVersion, imageCrop, containingFrameOpacity, boundTextCanvas }`. Used for cache-invalidation decisions.
- **`cappedElementCanvasSize(element, elementsMap, zoom): { width; height; scale }`** — L149-L202. Computes offscreen canvas pixel size and a draw scale, clamped to browser limits: `AREA_LIMIT = 16777216` (16M px), `WIDTH_HEIGHT_LIMIT = 32767`. Uses abs bounds via `getElementAbsoluteCoords`; for linear/freedraw uses `distance(x1,x2)`/`distance(y1,y2)` (rotated-aware extents) instead of `element.width/height`. Base size = `dim*devicePixelRatio + padding*2`. Scale starts at `zoom.value`; rescaled down if width/height×scale exceeds WIDTH_HEIGHT_LIMIT (L186-L191) or if area×scale² exceeds AREA_LIMIT via `sqrt(AREA_LIMIT/(w*h))` (L194-L196). Floors final dims. PERF/parity-critical: ties raster resolution to DPR×zoom.
- **`generateElementCanvas(element, elementsMap, zoom, renderConfig, appState): ExcalidrawElementWithCanvas | null`** — L204-L339. Creates an offscreen canvas sized by `cappedElementCanvasSize`; returns null if 0 width/height. For linear/freedraw, computes `canvasOffsetX/Y` = positive distance between `element.x/y` and bounds min (×DPR×scale) and translates context so negative-direction points fit (L228-L245); default offset is `(-100, 0)`. Translates by `padding*scale`, scales by `devicePixelRatio*scale`, then `drawElementOnCanvas` via a rough canvas (L247-L256). For arrows with a bound text label (L264-L321): builds a SQUARE `boundTextCanvas` of side `maxDim*DPR*scale + padding*scale*10` so rotation doesn't clip, draws the arrow canvas centered+rotated by `element.angle`, then rotates back, translates to the bound-text center, and `clearRect`s the label area (BOUND_TEXT_PADDING-expanded) so the unrotated label can be drawn separately. Returns the cache record including `boundTextElementVersion`, `containingFrameOpacity`, `imageCrop`. Side effect: allocates DOM canvases.
- **`const DEFAULT_LINK_SIZE = 14`** — L341 (exported constant).
- **`IMAGE_PLACEHOLDER_IMG` / `IMAGE_ERROR_PLACEHOLDER_IMG`** — L343-L359. Module-level `<img>` elements with inline-SVG data URIs (gray image / error glyph), guarded for non-browser (SSR) with a mock `{src:""}`.
- **`drawImagePlaceholder(element, context, theme)`** — L361-L385. Fills element rect with theme-dependent gray (#2E2E2E dark / #E7E7E7 light) and centers the placeholder/error img at `min(minDim, min(minDim*0.4, 100))` size.
- **`drawElementOnCanvas(element, rc, context, renderConfig)`** — L387-L601. Throws on unimplemented types (L597).
- **`const elementWithCanvasCache = new WeakMap<ExcalidrawElement, ExcalidrawElementWithCanvas>()`** — L603-L606 (exported). Per-element rasterization cache; keyed by element identity (regenerated on mutation since elements are immutable copies).
- **`generateElementWithCanvas(element, elementsMap, renderConfig, appState)`** — L608-L663. Cache wrapper: returns cached record unless zoom changed (and not `shouldCacheIgnoreZoom`), theme changed, `boundTextElementVersion` changed, `imageCrop` changed, `containingFrameOpacity` changed, or (arrow with bound text and angle changed — because the cached arrow canvas is reused under rotation but labels must stay unrotated). Computes element center in device px including `scrollX/Y` (L676-L677). Scales context by `1/DPR` (since cache is at DPR). Arrow-with-bound-text branch translates to center and draws the `boundTextCanvas` with offset compensation (L684-L700). General branch: translate to center, `rotate(element.angle)`, apply image `scale[0]/scale[1]` (only when image and not pending) (L708-L716), translate back, then `drawImage` at `(x1+scrollX)*DPR - padding` scaled by `1/scale` (L721-L729). Optional debug bound-text bounding-box stroke gated by `VITE_APP_DEBUG_ENABLE_TEXT_CONTAINER_BOUNDING_BOX` (L731-L749). Coordinate-space note: all positioning in device px; rotation origin is element center.
- **`renderSelectionElement(element, context, appState, selectionColor)`** — L756-L778 (exported). Draws the marquee/selection rect: translucent blue fill `rgba(0,0,200,0.04)`, 1px (zoom-adjusted) stroke in `selectionColor`, offset by `0.5/zoom.value` for crisp 1px lines.
- **`renderElement(element, elementsMap, allElementsMap, rc, context, renderConfig, appState)`** — L780-L1072 (exported). Top-level dispatcher. Sets `context.globalAlpha` from `getRenderOpacity`, reducing alpha to `DEFAULT_REDUCED_GLOBAL_ALPHA` when an elementLinkSelector dialog is open and the element is neither selected nor hovered (L789-L800). frame/magicframe → draws outline rect (round if `FRAME_STYLE.radius`), magic frames get a teal stroke (L803-L845). freedraw and the shape/text/image/arrow group each branch on `renderConfig.isExporting`: when exporting, draws directly with manual center-translate + rotate + shift (so no cache), with a dedicated arrow-with-bound-text temp-canvas path that scales by `appState.exportScale` and clears the label area (L890-L985); when not exporting, uses `generateElementWithCanvas` + `drawElementFromCanvas`, disabling `imageSmoothingEnabled` for angle 0 or right angles to sharpen (esp. text) (L1000-L1021), and renders a 10%-opacity uncropped ghost when cropping the active image (L1023-L1050). Resets `globalAlpha = 1` at end (L1071). Throws on unimplemented type.
- **`getFreedrawOutlineAsSegments(element, points, elementsMap): LineSegment<GlobalPoint>[]`** — L1074-L1128 (exported function). Builds a sequence of rotated world-space `lineSegment`s from a freedraw outline. Uses non-rotated bounds (`angle:0`) to find the rotation center, then for each point translates by `element.x/y` and applies `pointRotateRads(..., center, element.angle)`. `invariant(points.length >= 2)`. Reduce seeds with the first segment (points[0]→points[1]) then chains from `points.slice(2)`, each new segment starting at the previous segment's end. Used for eraser/collision against the freehand outline.

### packages/element/src/resizeElements.ts

The resize/rotate transform engine: converts a pointer position + active transform handle into new element geometry for single and multiple selections, handling aspect-ratio lock, resize-from-center, flipping, linear-point rescaling, bound-text font scaling, image scale flips, and elbow-arrow fixed-point mirroring.

- **`transformElements(originalElements, transformHandleType, selectedElements, scene, shouldRotateWithDiscreteAngle, shouldResizeFromCenter, shouldMaintainAspectRatio, pointerX, pointerY, centerX, centerY): boolean`** — L87-L201 (exported). Entry point; returns true if a transform happened. Single selection: "rotation" → `rotateSingleElement` (skips elbow arrows) + `updateBoundElements`; else computes `getNextSingleWidthAndHeightFromPointer` then `resizeSingleElement`; text elements get `updateBoundElements`. Multi selection: "rotation" → `rotateMultipleElements`; else `getNextMultipleWidthAndHeightFromPointer` then `resizeMultipleElements`. Side effects: mutates scene via `scene.mutateElement`.
- **`rotateSingleElement(element, scene, pointerX, pointerY, shouldRotateWithDiscreteAngle)`** — L203-L266. Computes center from abs coords; angle = `(5π/2 + atan2(pointerY-cy, pointerX-cx))` (the `5π/2` offset makes 0 = up). Discrete snapping rounds to `SHIFT_LOCKING_ANGLE` increments (adds half then subtracts modulo). `normalizeRadians`. Frame-likes forced to angle 0. Unbinds arrow start/end bindings if a binding element. Mutates element angle; also rotates the bound text (matching angle + repositioned via `computeBoundTextPosition`) except for arrows.
- **`rescalePointsInElement(element, width, height, normalizePoints)`** — L268-L283 (exported). For linear/freedraw, returns `{ points }` rescaled along x to `width` then y to `height` via `rescalePoints`; else `{}`.
- **`measureFontSizeFromWidth(element, elementsMap, nextWidth): { size } | null`** — L285-L308 (exported). Returns proportional font size = `fontSize * (nextWidth / width)` (width = bound-text max width if contained). Returns null if below `MIN_FONT_SIZE`.
- **`resizeSingleTextElement(origElement, element, scene, transformHandleType, shouldResizeFromCenter, nextWidth, nextHeight)`** — L310-L402 (exported). N/S handles scale font by height (width derived proportionally), repositions origin via `getResizedOrigin`. E/W handles re-wrap text to `max(minWidth, nextWidth)`, re-measure height, set `autoResize:false`. Mutates element.
- **`rotateMultipleElements(originalElements, elements, scene, pointerX, pointerY, shouldRotateWithDiscreteAngle, centerX, centerY)`** — L404-L488. Rotates each non-frame element around the GROUP center `(centerX, centerY)`: computes a group `centerAngle` (same `5π/2 + atan2` convention, optional discrete snap), rotates each element's own center by `(centerAngle + origAngle - element.angle)` to get its new center, then sets `x/y` shifted by `(rotatedC - c)` and `angle = normalize(centerAngle + origAngle)`. Elbow arrows instead just re-route via `getArrowLocalFixedPoints`. Unbinds bindings whose target isn't in the rotated set. Repositions bound text. Calls `scene.triggerUpdate()` once at end (batched).
- **`getResizeOffsetXY(transformHandleType, selectedElements, elementsMap, x, y): [number, number]`** — L490-L547 (exported). Computes the pointer offset (in rotated element space) from the relevant handle/edge, used to seed resize. De-rotates the pointer around the bbox center by `-angle`, then per-handle returns the rotated delta from the corresponding corner/edge midpoint.
- **`getResizeArrowDirection(transformHandleType, element): "origin" | "end"`** — L549-L560 (exported). For a 2-point linear element, decides whether the grabbed corner corresponds to resizing the origin or end based on sign of the second point and handle quadrant.
- **type `ResizeAnchor`** — L562-L571. Union: corners, sides (`west/north/east/south-side`), `center`.
- **`getResizeAnchor(handleDirection, shouldMaintainAspectRatio, shouldResizeFromCenter): ResizeAnchor`** — L573-L612. Maps handle → anchor (opposite corner/edge). `shouldResizeFromCenter` → "center". With aspect-ratio lock, side handles anchor to the opposite SIDE; corners anchor to the opposite corner. Without lock, e/se/s → top-left, n/nw/w → bottom-right, ne → bottom-left, else top-right.
- **`getResizedOrigin(prevOrigin, prevWidth, prevHeight, newWidth, newHeight, angle, handleDirection, shouldMaintainAspectRatio, shouldResizeFromCenter): { x; y }`** — L614-L720. KEY MATH for parity: given a resize anchored at one corner/edge/center while the element is rotated by `angle`, computes the new top-left so the anchor stays put. Each anchor case has a closed-form `x/y` expression mixing `(prevW-newW)/2`, `(prevH-newH)/2` with `cos(angle)`/`sin(angle)` terms (e.g. top-left at L634-L646). Center case is rotation-independent: `x - (newW-prevW)/2, y - (newH-prevH)/2`. This is the rotation-aware origin compensation a Canvas reimplementation must reproduce exactly.
- **`resizeSingleElement(nextWidth, nextHeight, latestElement, origElement, originalElementsMap, scene, handleDirection, { shouldInformMutation=true, shouldMaintainAspectRatio=false, shouldResizeFromCenter=false } = {})`** — L722-L926 (exported). Delegates text to `resizeSingleTextElement`. Bound text: with aspect-ratio lock, scales font via `measureFontSizeFromWidth` on the container's new max width (bails if null); without lock, clamps `nextWidth/Height` to approx min line width/height. Rescales linear points (`normalizePoints=true`) and re-derives origin offset from linear bounds (L806-L842). Negative width/height flips origin by adding the (negative) dimension (L845-L850). Image `scale` flip sign handling (L852-L860). Arrow bound-text font scaling under aspect lock (L862-L873). Only mutates if dims nonzero and origin finite; uses `Math.abs(width/height)`, attaches rescaled points, nulls/unbinds bindings (L888-L905), then `scene.mutateElement` with `{ informMutation, isDragging:false }`, updates bound-text font, calls `handleBindTextResize` and `updateBoundElements`.
- **`getNextSingleWidthAndHeightFromPointer(latestElement, origElement, handleDirection, pointerX, pointerY, { shouldMaintainAspectRatio, shouldResizeFromCenter } = {}): { nextWidth; nextHeight }`** — L928-L1020. Computes new dims from cursor. Uses `getResizedElementAbsoluteCoords` for orig and latest bounds; de-rotates pointer around START center by `-origElement.angle`. Per-axis scale from rotated pointer vs the start corner; `nextWidth = latestElement.width * scaleX` (uses ON-SCREEN dims so linear scaling matches the cursor). Resize-from-center: `2*next - orig`. Aspect-ratio lock: single-letter handles cross-multiply width/height ratios; 2-letter handles take `max(widthRatio,heightRatio)` preserving sign. Note: result dims can be negative (flip) — that's handled downstream.
- **`getNextMultipleWidthAndHeightFromPointer(selectedElements, originalElementsMap, elementsMap, handleDirection, pointerX, pointerY, { shouldMaintainAspectRatio, shouldResizeFromCenter } = {})`** — L1022-L1147. Builds the common bounding box of ORIGINAL elements plus manually-resolved bound-text positions (because `getCommonBoundingBox` returns null bound text for pointerDownState copies, L1041-L1068). `anchorsMap` gives the opposite corner/edge per handle (L1078-L1087); anchor = center if resize-from-center. Computes `nextWidth/Height` from `|pointer - anchor|` (×2 for center). `flipConditionsMap` (L1117-L1134) decides per-axis flip booleans by which side of the anchor the pointer crossed. Returns `{ originalBoundingBox, nextWidth, nextHeight, flipByX, flipByY }`.
- **`resizeMultipleElements(selectedElements, elementsMap, handleDirection, scene, originalElementsMap, { shouldMaintainAspectRatio, shouldResizeFromCenter, flipByX, flipByY, nextHeight, nextWidth, originalBoundingBox } = {})`** — L1149-L1500 (exported). Resizes/flips a group about a common anchor. Bails on all-undefined or zero dims. Recomputes `originalBoundingBox` if not passed (including bound text). Computes per-axis `scaleX/scaleY` (or uniform `scale` for corners / when `keepAspectRatio`). `keepAspectRatio` is forced if any element is rotated, is text, or is in a group (L1306-L1313) — important parity rule. Flip factors `[flipByX?-1:1, flipByY?-1:1]`. For each non-bound-text element computes new `width/height = orig.dim*scale`, flipped `angle = normalize(orig.angle*flipFactorX*flipFactorY)`, and new `x/y = anchor + flipFactor*(offset*scale + shift)` where `shift` is width/height for non-linear flips (linear/freedraw mirror via `rescalePointsInElement` with signed dims). Elbow arrows mirror `startBinding/endBinding.fixedPoint` (`-fp+1`) and remap `fixedSegments` to rescaled points (L1382-L1418). Images flip `scale` sign. Text recomputes font size. Bound text font scaled by `scale` under aspect lock. Applies all updates in a loop with `updateBoundElements`, unbinds out-of-set bindings, then a single `scene.triggerUpdate()`.

- **`const RESIZE_CURSORS = ["ns", "nesw", "ew", "nwse"]`** — L219.
