## Cluster: excalidraw__scene__0

This cluster contains the scene-level orchestration layer of the Excalidraw editor: image/SVG/canvas export, viewport scroll math, scrollbar geometry, zoom/grid normalization, the memoized renderable-element selector, and the render-config type definitions. These are the modules a Svelte/Canvas reimplementation must match for export fidelity and viewport/scroll/scrollbar parity.

### packages/excalidraw/scene/export.ts

Produces export artifacts (raster canvas, SVG) from a set of elements + app state, including frame-label injection, frame clip paths, font inlining, and an embedded-scene base64 payload round-trip.

- `truncateText(element: ExcalidrawTextElement, maxWidth: number)` → `ExcalidrawTextElement` (L65-L94). If the text element already fits within `maxWidth`, returns it unchanged. Otherwise it creates an offscreen `<canvas>` 2D context, sets the font via `getFontString`, measures the full text, and if it overflows, iterates **from the right** (`for i = text.length; i > 0; i--`) trying `text.slice(0,i) + "..."` until it fits, then returns `newElementWith(element, { text, width: maxWidth })`. Side effect: creates a DOM canvas + 2D context for measurement. Performance note: the right-to-left shrink is an intentional optimization assuming frame names overflow by only a few characters (comment L81-L84). Math/coord note: measurement is in CSS pixels of the unscaled font.
- `addFrameLabelsAsTextElements(elements: readonly NonDeletedExcalidrawElement[], opts: Pick<AppState, "exportWithDarkMode">)` → `NonDeletedExcalidrawElement[]` (L102-L131). For each element, if it is a frame-like element it synthesizes a temporary `newTextElement` for the frame's title using `FRAME_STYLE` constants (`nameOffsetY`, `nameFontSize`, `nameLineHeight`, light/dark name color), positioned at `x = element.x`, `y = element.y - FRAME_STYLE.nameOffsetY` then shifted up by the text element's own height (`textElement.y -= textElement.height`), truncated to `element.width`, and pushed before the frame itself. Coordinate note: the label sits above the frame's top edge; this is a DOM-to-canvas hack documented at L96-L101 (frame labels are normally DOM-rendered while editing).
- `getFrameRenderingConfig(exportingFrame: ExcalidrawFrameLikeElement | null, frameRendering: AppState["frameRendering"] | null)` → `AppState["frameRendering"]` (L133-L144). Falls back to `getDefaultAppState().frameRendering` when null. When a specific frame is being exported it forces `enabled:true, outline:false, name:false, clip:true`; otherwise it passes through the supplied flags. Invariant: exporting a single frame always clips and never draws outline/name.
- `prepareElementsForRender({ elements, exportingFrame, frameRendering, exportWithDarkMode })` → `readonly ExcalidrawElement[]` (L146-L174). Three-way branch: if `exportingFrame`, returns `getElementsOverlappingFrame(elements, exportingFrame, arrayToMap(elements))`; else if `frameRendering.enabled && frameRendering.name`, returns `addFrameLabelsAsTextElements(...)`; else returns `elements` unchanged.
- `exportToCanvas(elements, appState, files, { exportBackground, exportPadding = DEFAULT_EXPORT_PADDING, viewBackgroundColor, exportingFrame }, createCanvas?, loadFonts?)` → `Promise<HTMLCanvasElement>` (L176-L280). Exported. Async. Awaits `loadFonts()` (default loads element fonts via `Fonts.loadElementsFonts`). Builds frame rendering config; when `exportingFrame` set, forces `clip=false` (so frame corners aren't clipped, comment L211-L212) and `exportPadding=0`. Computes bounds via `getCanvasSize` over `[exportingFrame]` or `getRootElements(elementsForRender)`. Calls `createCanvas(width,height)` (default makes a `<canvas>` sized `width*appState.exportScale × height*appState.exportScale`, returning `scale`). Builds image cache via `updateImageCache`. Calls `renderStaticScene` with a derived appState whose `scrollX = -minX + exportPadding`, `scrollY = -minY + exportPadding`, `zoom = defaultAppState.zoom` (i.e. zoom reset to 1), `shouldCacheIgnoreZoom:false`, theme from `exportWithDarkMode`, and renderConfig `{ renderGrid:false, isExporting:true, embedsValidationStatus:empty (disables embeddables), elementsPendingErasure:empty, pendingFlowchartNodes:null }`. Coordinate note: scroll offsets translate scene-space min corner to padding origin. Returns the canvas (NOT a blob).
- `createHTMLComment(text: string)` → `Comment` (L282-L287). Returns `document.createComment(" text ")` with surrounding spaces to keep prettified output stable vs. older versions.
- `exportToSvg(elements, appState, files, opts?)` → `Promise<SVGSVGElement>` (L289-L506). Exported. Builds frame rendering config and destructures export options. Computes `elementsForRender`, then `getCanvasSize`, then `offsetX = -minX + exportPadding`, `offsetY = -minY + exportPadding`. Creates `<svg>` (namespace `SVG_NS`) with `version="1.1"`, `viewBox="0 0 width height"`, `width = width*exportScale`, `height = height*exportScale`, plus `<defs>` and `<metadata>`. If `exportEmbedScene`, embeds the **original** (pre-frame-label) elements as base64 via `encodeSvgBase64Payload(serializeAsJSON(...))` (comment L372-L373 stresses original elements). Frame clip paths (L393-L429): for each frame-like element builds a `<clipPath id=frame.id>` containing a `<rect>` transformed by `translate(frame.x+offsetX frame.y+offsetY) rotate(frame.angle cx cy)` where `cx=(x2-x1)/2-(frame.x-x1)`, `cy=(y2-y1)/2-(frame.y-y1)` (rotation pivot in element-local space derived from `getElementAbsoluteCoords`); adds `rx/ry=FRAME_STYLE.radius` unless exporting a single frame. Inlines `@font-face` declarations via `Fonts.generateFontFaceDeclarations` joined by a 6-space delimiter into a `<style class="style-fonts">` (skipped if `opts.skipInliningFonts`). Adds a background `<rect>` covering the full size with `applyDarkModeFilter` color when dark mode. Renders elements via `renderSceneToSvg` passing `{ offsetX, offsetY, isExporting:true, renderEmbeddables (default false), embedsValidationStatus (maps frame ids→true when rendering embeddables), reuseImages (default true), theme }`. Geometry note: the clipPath rotation pivot math is load-bearing for rotated frames.
- `encodeSvgBase64Payload({ payload, metadataElement })` → `void` (L508-L527). Exported. Base64-encodes `JSON.stringify(encode({ text: payload }))` (with `isByteString=true`) and appends to the metadata element a sequence of HTML comments: `payload-type:<MIME_TYPES.excalidraw>`, `payload-version:2`, `payload-start`, then a text node with the base64, then `payload-end`. Side effect: mutates `metadataElement`.
- `decodeSvgBase64Payload({ svg })` → `string | <decoded>` (L529-L561). Exported. Only proceeds if the SVG string contains `payload-type:<excalidraw>`; matches the base64 between `<!-- payload-start -->` and `<!-- payload-end -->` (regex, non-greedy), reads `payload-version` (default "1"), treats `version !== "1"` as a byte string. Decodes base64→json; if the parsed object lacks an `"encoded"` key it returns the raw json only when it is a legacy un-encoded excalidraw scene (`type === EXPORT_DATA_TYPES.excalidraw`), otherwise throws `"FAILED"`; if it has `"encoded"` it returns `decode(encodedData)`. Throws `"INVALID"` when no payload present. Invariant: version handling distinguishes v1 (string) from v2 (byte string) base64 encoding.
- `getCanvasSize(elements: readonly NonDeletedExcalidrawElement[], exportPadding: number)` → `Bounds` (L564-L573). Internal. Computes `getCommonBounds` → `[minX,minY,maxX,maxY]`, then `width = distance(minX,maxX) + exportPadding*2`, `height = distance(minY,maxY) + exportPadding*2`, returns `[minX, minY, width, height]`. Note: returns the min corner plus padded dimensions (NOT max corner).
- `getExportSize(elements, exportPadding, scale)` → `[number, number]` (L575-L585). Exported. Calls `getCanvasSize`, takes the `width`/`height` entries, multiplies by `scale` and `Math.trunc`s each, returns `[width, height]`. Note: maps over the full Bounds tuple but only destructures indices 2,3.

### packages/excalidraw/scene/Renderer.ts

Defines the `Renderer` class: a memoized selector that computes the renderable elements map and the viewport-visible element list for the static canvas, with special handling for in-progress new elements, text being edited, and dragging elements into a highlighted frame.

- `class Renderer` (L40-L262). Exported. Holds `private scene: Scene`.
- `constructor(scene: Scene)` (L43-L45). Stores the scene reference.
- `private getVisibleCanvasElements({ elementsMap, zoom, offsetLeft, offsetTop, scrollX, scrollY, height, width })` → `readonly NonDeletedExcalidrawElement[]` (L47-L87). Iterates `elementsMap.values()`, keeps each element for which `isElementInViewport(element, width, height, { zoom, offsetLeft, offsetTop, scrollX, scrollY }, elementsMap)` is true. Output: viewport-culled element list. Coordinate note: viewport test uses scroll + zoom + offset to map scene to screen.
- `private getRenderableElementsMap({ elements, editingTextElement, newElement })` → `{ elementsMap, newElementCanvasElement }` (L89-L117). Builds a branded `RenderableElementsMap`. Sets `newElementCanvasElement = newElement?.frameId ? null : newElement` (a new element being placed *inside a frame* is not drawn on the static layer). Skips the element whose id equals `newElementCanvasElement.id`, and skips the text element currently being edited (`editingTextElement.type === "text" && element.id === editingTextElement.id`) because it is rendered on the remote/overlay layer (comment L106-L107). Invariant: editing text + new-element-in-frame are excluded from the static map.
- `private sortSelectedElementsIntoHighlightedFrame<T extends ExcalidrawElement>({ visibleElements, selectedElements, frameToHighlight })` → `readonly T[]` (L119-L157). If no selected elements, returns `visibleElements` unchanged. Builds a map of selected element ids; computes `deselectedElements` (visible minus selected). Calls `getFrameChildrenInsertionIndex(deselectedElements, frameToHighlight.id)`; if null returns input unchanged; otherwise returns `[...deselected.slice(0,idx), ...selectedElements, ...deselected.slice(idx)]`. Purpose: visually reorders dragged-selection into z-position they'd occupy inside the highlighted frame, *without* committing `frameId` (comment L230-L234 explains commit deferred to pointerup).
- `private _getRenderableElements = memoize((opts & { canvasNonce }))` (L159-L206). Memoized selector keyed on `canvasNonce` + viewport params (zoom/offset/scroll/size) + `editingTextElement` + `newElement`. Pulls `this.scene.getNonDeletedElements()`, builds the renderable map, culls to visible, and returns `{ elementsMap, visibleElements, newElementCanvasElement, canvasNonce }`. Performance note: this is the core render-skip cache; `canvasNonce` invalidation drives re-computation. NOT keyed on selection/drag/frameToHighlight (those are applied post-memo).
- `public getRenderableElements = (opts: GetRenderableElementsOpts)` (L208-L254). Computes `canvasNonce = scene.getSceneNonce()` optionally suffixed with `:${newElement.versionNonce}` when the new element is inside a frame (so frame-bound new-element edits invalidate the cache). Calls the memoized selector with only the memo-relevant props (explicit pick, NOT spread, comment L217). After the memo, if `frameToHighlight && selectedElementsAreBeingDragged && getCommonFrameId(selectedElements) !== frameToHighlight.id`, it reorders `visibleElements` via `sortSelectedElementsIntoHighlightedFrame` and returns a copy; else returns the memoized result. Invariant: reorder only happens when dragging into a *different* frame than the elements' current common frame.
- `public destroy()` (L256-L261). Cancels `renderStaticSceneThrottled` and clears the memoized selector cache. Comment L256-L257 notes it intentionally does NOT destroy scene/rc to preserve the TS contract for upstream consumers.
- Type `GetRenderableElementsOpts` (L25-L38): zoom, offsetLeft, offsetTop, scrollX, scrollY, height, width, editingTextElement, newElement, selectedElements, selectedElementsAreBeingDragged, frameToHighlight.

### packages/excalidraw/scene/scroll.ts

Computes scroll offsets to center the viewport on a point or on the scene's elements, with viewport-overflow fallback.

- `isOutsideViewPort(appState: AppState, cords: Array<number>)` → `boolean` (L15-L29). Internal. Takes scene-space `[x1,y1,x2,y2]`, converts both corners to viewport coords via `sceneCoordsToViewportCoords`, returns true if the projected width `> appState.width` OR projected height `> appState.height`. Coordinate note: tests whether the elements bbox would overflow the viewport at current zoom.
- `centerScrollOn({ scenePoint, viewportDimensions, zoom, offsets? })` → `{ scrollX, scrollY }` (L31-L58). Exported. Computes `scrollX = (viewportWidth - offsets.right)/2/zoom.value - scenePoint.x`, then adds `offsets.left/2/zoom.value`; analogously for `scrollY` using bottom/top offsets. Math note: divides viewport halves by zoom to convert to scene units; `offsets` lets UI panels (left/right/top/bottom insets) bias the centering. Output is scene-space scroll.
- `calculateScrollCenter(elements: readonly ExcalidrawElement[], appState: AppState)` → `{ scrollX, scrollY }` (L60-L92). Exported. Filters to `getVisibleElements`; if none, returns `{0,0}`. Computes `getCommonBounds`; if the bbox is outside the viewport (`isOutsideViewPort`), it falls back to `getClosestElementBounds(elements, viewportCoordsToSceneCoords({ clientX: scrollX, clientY: scrollY }, appState))` (i.e. bounds nearest the current scroll-derived scene point). Then centers on the bbox midpoint `((x1+x2)/2,(y1+y2)/2)` via `centerScrollOn`. Coordinate note: when elements are too big to fit, it centers on the nearest element instead of the whole bbox.

### packages/excalidraw/scene/scrollbars.ts

Computes the geometry (position/size) of horizontal & vertical scrollbars representing where the viewport sits in the scene, plus hit-testing against them.

- Constants `SCROLLBAR_MARGIN = 4`, `SCROLLBAR_WIDTH = 6`, `SCROLLBAR_COLOR = "rgba(0,0,0,0.3)"` (L10-L12). Exported geometry/style constants for scrollbar rendering and hit-testing.
- `getScrollBars(elements: RenderableElementsMap, viewportWidth, viewportHeight, appState: InteractiveCanvasAppState)` → `ScrollBars` (L15-L130). Exported. Returns `{horizontal:null, vertical:null}` when no elements. Otherwise: computes elements bbox via `getCommonBounds`; zoom-adjusts viewport (`viewportWidthWithZoom = viewportWidth / zoom.value`, likewise height); reads CSS safe-area insets (`sat/sab/sal/sar`) via `getGlobalCSSVariable`; checks RTL via `getLanguage().rtl`. Computes the viewport bbox in scene coords (`viewportMinX = -scrollX + safeArea.left`, etc.), the union "scene" bbox (`min/max` of elements+viewport), the elements-only width/height, and the extended (union) width/height. Computes `scrollbarWidth = viewportWidth*(viewportWidthWithZoom/extendedSceneWidth) - scrollWidthOffset` (and the height analog), where the offsets reserve room for margins/safe-areas plus `SCROLLBAR_WIDTH*2`. Computes `horizontalDeltaMultiplier`/`verticalDeltaMultiplier` that translate scrollbar drag distance into scene scroll: when the viewport extends beyond the elements bbox it uses `(extendedScene*zoom)/(scrollbar+offset)`, otherwise `viewport/(scrollbar+offset)` (the comment L79-L81 explicitly flags an acknowledged small accumulation error in the extended case). Returns `horizontal:null` when `viewportMinX===sceneMinX && viewportMaxX===sceneMaxX` (i.e. nothing to scroll horizontally), else an object `{x,y,width,height,deltaMultiplier}` positioned at the bottom of the viewport; `vertical` analogously positioned at the right (or left when RTL) of the viewport. Coordinate/perf note: this is dense world↔screen math and the `deltaMultiplier` is the load-bearing value a reimplementation must match for scroll-by-scrollbar parity.
- `isOverScrollBars(scrollBars: ScrollBars, x: number, y: number)` → `{ isOverEither, isOverHorizontal, isOverVertical }` (L132-L155). Exported. Maps over `[horizontal, vertical]`, each true when non-null and the point `(x,y)` lies within `[scrollBar.x, scrollBar.x+width] × [scrollBar.y, scrollBar.y+height]`. `isOverEither` is the OR. Inputs are viewport (screen) coordinates.
