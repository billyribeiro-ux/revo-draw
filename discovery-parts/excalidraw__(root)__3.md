## Cluster: excalidraw__(root)__3

This cluster covers React batching helpers, keyboard-shortcut formatting, the object-snapping engine, the text auto-resize handle geometry, the central editor type surface, the Vite env typings, and the short-lived worker pool. Two files (`vite-env.d.ts`, plus the types-only `types.ts`) are declaration/type surfaces; `snapping.ts` is the algorithmically dense one and matters most for a Canvas/Svelte parity comparison.

### packages/excalidraw/reactUtils.ts

Helpers that wrap React-DOM batched updates and decide whether render throttling is allowed for the running React version.

- `withBatchedUpdates<TFunction>(func)` — **L10-L17.** Generic helper constrained so `func` takes 0 or 1 parameter (`Parameters<TFunction>["length"] extends 0 | 1 ? TFunction : never`). Returns a new handler `(event) => unstable_batchedUpdates(func, event)` cast back to `TFunction`. Side effect: forces all React state updates inside `func` into a single batch. Invariant: the handler must accept at most one argument (an event), enforced at the type level.
- `withBatchedUpdatesThrottled<TFunction>(func)` — **L23-L32.** Same single-arg constraint, but wraps the batched call in `throttleRAF` (from `@excalidraw/common`) so the handler fires at most once per animation frame. Uses a `@ts-ignore` (L28) to bridge the throttle generic. Output: a throttled, batched handler. Performance detail: collapses bursts of events (pointermove etc.) to one batched React update per rAF.
- `isRenderThrottlingEnabled` — **L34-L62.** An IIFE returning a predicate `() => boolean`. On init it parses `ReactVersion` (`react`'s `version` export), sets `IS_REACT_18_AND_UP = Number(major) > 17` inside a try/catch (defaults false on parse error). The returned closure returns `true` only when `window.EXCALIDRAW_THROTTLE_RENDER === true` AND React ≥ 18; on React < 18 it warns once (`hasWarned` latch, L45/L50) and returns false. Notable: references GitHub issue #5439 in comment (L35-L36) as the reason for disabling throttling on React < 18. Closure-captured state: `IS_REACT_18_AND_UP`, `hasWarned`.

### packages/excalidraw/shortcut.ts

Formats a raw shortcut descriptor string into platform/locale-appropriate key labels.

- `getShortcutKey(shortcut: string): string` — **L5-L19.** Pure string transform. Chains `.replace()` calls over regex tokens, swapping in localized labels via `t()` (i18n): `Opt`/`Option`/`Alt` → `keys.option` on Darwin else `keys.alt`; `Shift` → `keys.shift`; `Enter`/`Return` → `keys.enter`; `Ctrl`/`Cmd`/`Command`/`CtrlOrCmd` → `keys.cmd` on Darwin else `keys.ctrl` (global flag `gi`, so all occurrences); `Esc`/`Escape` → `keys.escape`; `Space`/`Spacebar` → `keys.spacebar`; `Del`/`Delete` → `keys.delete`. Platform branch uses `isDarwin` from `@excalidraw/common`. Notable: only the Ctrl/Cmd replace is global (`g`); the others replace the first match only. Word-boundary `\b` anchors avoid partial matches.

### packages/excalidraw/snapping.ts

The object-snapping engine: builds reference snap points/gaps from non-selected elements and computes the offset + visual snap lines used when dragging, resizing, creating, or pointer-hovering elements. **This is the parity-critical file.**

Module constants / types:
- `SNAP_DISTANCE = 8` (**L41**) — base snap threshold in screen px (before zoom division).
- `VISIBLE_GAPS_LIMIT_PER_AXIS = 99999` (**L45**) — cap on gap pairs computed per axis (currently effectively unbounded; TODO note at L43-L44).
- Type aliases: `Vector2D {x,y}` (L52), `PointPair = [GlobalPoint, GlobalPoint]` (L57), `PointSnap {type:"point"; points; offset}` (L59-L63), `Gap` (L65-L81, with ASCII diagram describing startBounds/endBounds/startSide/endSide/overlap/length), `GapSnap` with 6-way `direction` union (L83-L94), `GapSnaps`, `Snap = GapSnap|PointSnap`, `Snaps`, `PointSnapLine {type:"points"; points}` (L101-L104), `PointerSnapLine {type:"pointer"; points; direction}` (L106-L110), `GapSnapLine {type:"gap"; direction; points}` (L112-L116), `SnapLine` union (L118).

Functions / class:
- `getSnapDistance(zoomValue): number` — **L48-L50.** Returns `SNAP_DISTANCE / zoomValue` so the snap threshold stays constant in screen pixels regardless of zoom (world-space threshold shrinks as you zoom in). Coordinate-space detail: all snap math is in world space, hence the division.
- `class SnapCache` — **L122-L155.** Static-only cache holding `referenceSnapPoints: GlobalPoint[] | null` and `visibleGaps: {verticalGaps, horizontalGaps} | null`. Methods: `setReferenceSnapPoints` (L130), `getReferenceSnapPoints` (L134), `setVisibleGaps` (L138), `getVisibleGaps` (L147), `destroy` (L151, nulls both). Invariant/performance: the caller pre-populates this cache on pointer-down so per-move snap computation doesn't recompute reference geometry every frame.
- `isGridModeEnabled(app): boolean` — **L159-L160.** Returns `app.props.gridModeEnabled ?? app.state.gridModeEnabled` (prop overrides state).
- `isSnappingEnabled({event, app, selectedElements})` — **L162-L192.** Decides if snapping is active. With an `event` (keyboard modifiers): blocks during lasso *selection* but allows when lasso-dragging selected elements; otherwise snaps when `objectsSnapModeEnabled && !CTRL_OR_CMD`, OR (inverse) `!objectsSnapModeEnabled && CTRL_OR_CMD && !gridMode` (Ctrl temporarily toggles snapping). Without an event: single selected arrow returns `false` (defer to binding), else returns `objectsSnapModeEnabled`. Key behavioral invariant for parity: Ctrl/Cmd inverts the snap mode.
- `areRoughlyEqual(a, b, precision = 0.01): boolean` — **L194-L196.** `Math.abs(a-b) <= precision`. Used for angle≈0 checks.
- `getElementsCorners(elements, elementsMap, {omitCenter, boundingBoxCorners, dragOffset})` — **L198-L313.** Computes candidate snap points. Single element (L216-L292): gets absolute coords `[x1,y1,x2,y2,cx,cy]`, applies `dragOffset` to all, computes half-width/half-height. For diamond/ellipse (unless `boundingBoxCorners`) it uses the 4 edge midpoints (left/top/right/bottom mid) + optional center; otherwise the 4 rotated rectangle corners + optional center. All corners are rotated about `(cx,cy)` by `element.angle` via `pointRotateRads`. Multi-element (L293-L310): uses `getDraggedElementsBounds` AABB → 4 corners + center (no rotation). Returns every point rounded via `round()` (L312, 6-decimal). Geometry/parity note: ellipse/diamond snap to edge midpoints not corners; rotation is applied per-point about the element center.
- `getReferenceElements(elements, selectedElements, appState, elementsMap)` — **L315-L326.** Thin wrapper over `getVisibleAndNonSelectedElements` — the set of elements snaps can target.
- `getVisibleGaps(elements, selectedElements, appState, elementsMap)` — **L328-L444.** Builds horizontal+vertical equal-gap candidates. Computes `referenceBounds` from `getMaximumGroups` (excludes single bound-to-container elements, L342-L345), rounds bounds. Sorts by `[0]` (minX) for horizontal pass (L353): for each ordered pair `(i,j)` where `startMaxX < endMinX` and their Y ranges overlap (`rangesOverlap`), pushes a `Gap` with `startSide`/`endSide` vertical segments, `length = endMinX - startMaxX`, and `overlap = rangeIntersection` of Y ranges (L359-L395). Then sorts by `[1]` (minY) for the vertical pass with symmetric logic on X ranges (L397-L438). Both loops respect the `VISIBLE_GAPS_LIMIT_PER_AXIS` counter `c` with labeled `break` (L363/L407). **Parity gotcha:** `referenceBounds` is sorted in place twice — the horizontal pass sorts by minX, the vertical pass re-sorts the same array by minY (L353 vs L397). Returns `{horizontalGaps, verticalGaps}`.
- `getGapSnaps(selectedElements, dragOffset, app, event, nearestSnapsX, nearestSnapsY, minOffset)` — **L446-L614.** For the dragged selection's AABB (rounded), iterates cached `horizontalGaps`: skips gaps whose `overlap` doesn't intersect the selection's Y range. Computes three candidate snaps per gap — center (selection centered in gap, only if gap longer than selection, L483), side-from-right (L502-L521), side-from-left (L523-L542) — each compared against `minOffset.x`; when a strictly-smaller offset is found it clears `nearestSnapsX` (`.length = 0`) and updates `minOffset.x`. Symmetric vertical logic for `verticalGaps` (center_vertical L549, side_top L571, side_bottom L592). Mutates `nearestSnapsX/Y` and `minOffset` in place (output via side effect). Invariant: `<=` keeps ties, `<` resets — so equal-distance snaps accumulate.
- `getReferenceSnapPoints(elements, selectedElements, appState, elementsMap)` — **L616-L634.** Flattens `getElementsCorners` over each maximum group of reference elements (again excluding single bound-to-container elements). Output cached into `SnapCache.referenceSnapPoints`.
- `getPointSnaps(selectedElements, selectionSnapPoints, app, event, nearestSnapsX, nearestSnapsY, minOffset)` — **L636-L690.** Nested loop over every selection snap point × every cached reference snap point; computes `offsetX/offsetY` deltas; for each axis, if `|offset| <= minOffset`, push a `PointSnap` (clearing the array on strict `<`) and tighten `minOffset`. Pure side-effect mutation of the four accumulators.
- `snapDraggedElements(elements, dragOffset, app, event, elementsMap)` — **L692-L807.** Top-level drag-snap. Two-pass design: (1) rounds `dragOffset`, seeds `minOffset` with `getSnapDistance(zoom)`, runs `getPointSnaps`+`getGapSnaps` to find `snapOffset = {nearestSnapsX[0]?.offset ?? 0, ...Y}` (L751-L754); (2) resets accumulators + minOffset to 0, recomputes snaps at the *snapped* position `newDragOffset = round(dragOffset + snapOffset)` so the rendered snap lines are exact (comment L756-L761). Builds `pointSnapLines` (`createPointSnapLines`) and `gapSnapLines` (`createGapSnapLines`, filtering to gap snaps). Returns `{snapOffset, snapLines}`. **Parity-critical two-phase: compute offset, then recompute lines at the snapped position to avoid line jitter.**
- `round(x): number` — **L809-L812.** `Math.round(x * 1e6) / 1e6` — 6-decimal rounding used pervasively for stable dedupe keys and float hygiene.
- `dedupePoints(points): GlobalPoint[]` — **L814-L826.** Dedupes by `point.join(",")` string key via a Map, preserving first occurrence.
- `createPointSnapLines(nearestSnapsX, nearestSnapsY): PointSnapLine[]` — **L828-L896.** Groups point snaps by shared X coordinate (`round(snap.points[0][0])`) and by shared Y coordinate; for each X-group emits a vertical line of points sorted by Y, for each Y-group a horizontal line sorted by X; dedupes points. Output: collinear snap guides.
- `dedupeGapSnapLines(gapSnapLines): GapSnapLine[]` — **L898-L913.** Dedupes gap lines by a flattened, rounded points key.
- `createGapSnapLines(selectedElements, dragOffset, gapSnaps): GapSnapLine[]` — **L915-L1106.** For each `GapSnap`, computes the intersection of the selection's range with `gap.overlap` to place the guide line at the overlap midpoint, then emits a pair of segments per direction (`center_horizontal` L943, `center_vertical` L969, `side_right` L995, `side_left` L1018, `side_top` L1044, `side_bottom` L1070). Final map rounds all points and dedupes. Geometry detail: guide lines are drawn at the midpoint of the overlapping span between the two reference elements.
- `snapResizingElements(selectedElements, selectedOriginalElements, app, event, dragOffset, transformHandle)` — **L1108-L1244.** Resize-time snapping. Bails if disabled, empty, or a single rotated element (`!areRoughlyEqual(angle, 0)`, L1121-L1122). Applies `dragOffset` to the relevant side(s) of the original common bounds depending on which handle (`e/w/n/s`, L1132-L1144), then builds `selectionSnapPoints` per handle type (edges contribute two corner points, corners contribute one, L1148-L1183). Two-pass like the drag path: find `snapOffset`, then recompute point snap lines from the *current* common bounds corners (L1217-L1238). Returns `{snapOffset, snapLines}`. Note: only point snaps (no gap snaps) on resize.
- `snapNewElement(newElement, app, event, origin, dragOffset, elementsMap)` — **L1246-L1316.** Snapping while creating a new element. First pass snaps the single pointer point `origin + dragOffset`; second pass recomputes lines from the element's bounding-box corners (`boundingBoxCorners: true, omitCenter: true`). Returns `{snapOffset, snapLines}`.
- `getSnapLinesAtPointer(elements, app, pointer, event, elementsMap)` — **L1318-L1400.** Pointer-hover snapping (e.g. before drawing). Iterates every reference element's corners; tracks the nearest vertical (matching X) and horizontal (matching Y) `PointerSnapLine`. **Subtle bug-prone detail:** `minOffset` here stores the *signed* offset, and comparisons use `Math.abs(minOffset.x)` (L1355) — unlike the other paths which store absolute offsets. Returns `{originOffset, snapLines}` where `originOffset` is the delta to snap the pointer to the nearest corner.
- `isActiveToolNonLinearSnappable(activeToolType): boolean` — **L1402-L1414.** True for rectangle/ellipse/diamond/frame/magicframe/image/text tools (the shape-creation tools that support snapping while drawing).

### packages/excalidraw/textAutoResizeHandle.ts

Geometry + hit-testing for the small drag handle (desktop only) that toggles text auto-resize, positioned to the right of a text element.

Module constants — **L13-L18:** `TEXT_AUTO_RESIZE_HANDLE_GAP = 12`, `TEXT_AUTO_RESIZE_HANDLE_LENGTH = 16`, `TEXT_AUTO_RESIZE_HANDLE_HITBOX_WIDTH = 10`, `TEXT_AUTO_RESIZE_HANDLE_HITBOX_HEIGHT = LENGTH + 2 (=18)`, `MAX_HANDLE_HEIGHT_RATIO = 0.8`.

- `getTextBoxPadding(zoomValue): number` — **L20-L21.** `(DEFAULT_TRANSFORM_HANDLE_SPACING * 2) / zoomValue` — padding in world units, constant on screen.
- `getTextAutoResizeHandle(textElement, zoomValue, formFactor)` — **L23-L64.** Returns the handle geometry, or `null` if not desktop, or if the handle length would exceed `height * zoom * 0.8` (handle hidden on too-short text, L28-L34). Computes element center, then `handleCenter` = a point `width/2 + padding + gap` to the right of center, rotated about center by `element.angle` (L43-L47). Returns `{center, start, end, hitboxWidth, hitboxHeight}` where `start`/`end` are the handle segment endpoints (`handleCenter.y ∓ length/2`) themselves rotated about `handleCenter` by the angle, and hitbox dims are divided by zoom (L49-L63). Coordinate detail: gap/length/padding are all `/zoomValue` (screen-constant); the segment is rotated twice — once into the right-offset position, once for the segment endpoints.
- `isPointHittingTextAutoResizeHandle(point, textElement, zoomValue, formFactor): boolean` — **L66-L88.** Calls `getTextAutoResizeHandle`; if null → false. Otherwise un-rotates the test point about `handle.center` by `-element.angle` (L78-L82), then does an axis-aligned half-extent box test against `hitboxWidth/2` and `hitboxHeight/2` (L84-L87). Geometry detail: inverse-rotation reduces the rotated-rectangle hit test to an AABB test — standard OBB technique.

### packages/excalidraw/workers.ts

A pool of idle, short-lived module Workers (used e.g. for font subsetting) that auto-terminate after a TTL of inactivity.

- `class IdleWorker` — **L5-L16.** Wraps a `Worker` (`new Worker(workerUrl, {type:"module"})`, L8-L10). Holds a `debounceTerminate!: ReturnType<typeof debounce>` (definite-assignment, set later by the pool) used to prolong life or terminate-with-flush. Invariant: `debounceTerminate` is assigned by `WorkerPool.createWorker` before use.
- `class WorkerPool<T, R>` — **L23-L154.** Generic pool (`T` = message in, `R` = result out).
  - Private fields: `idleWorkers: Set<IdleWorker>`, `workerUrl: URL`, `workerTTL: number`. Constructor (private, L28-L37) defaults `ttl` to 1000ms. Comment (L18-L21) notes it intentionally does **not** cap worker count — caller manages pool size.
  - `static create<T,R>(workerUrl, options)` — **L47-L63.** Factory. Throws `WorkerUrlNotDefinedError` if no URL (L53-L55), and `WorkerInTheMainChunkError` if `workerUrl` equals `import.meta.url` (i.e. the worker got bundled into the main chunk, L57-L60). Returns a new pool. Invariant: prevents accidentally running the "worker" on the main thread.
  - `async postMessage(data, options: StructuredSerializeOptions): Promise<R>` — **L68-L93.** Reuses an idle worker (shift from the set, delete it) or creates one; wires `onmessage`/`onerror`, posts the message, and arms `debounceTerminate` with a reject callback that fires if the worker doesn't respond within `workerTTL` ("Active worker did not respond"). Returns a Promise of the worker's result. Side effects: removes worker from idle set while busy.
  - `async clear()` — **L98-L105.** Cancels each idle worker's debounce and terminates it, then empties the set. Used to flush the pool (also called on error).
  - `private async createWorker(): Promise<IdleWorker>` — **L110-L131.** Builds an `IdleWorker` and assigns its `debounceTerminate` = a `debounce(fn, workerTTL)` that terminates the worker; if the worker is still in the idle set it removes it and logs a debug release message, else if a `reject` was passed it rejects, else logs a "Worker has been terminated!" error (L113-L128). Performance/lifecycle detail: the same debounce both keeps an idle worker alive and enforces the response timeout.
  - `private onMessageHandler(worker, resolve)` — **L133-L139.** Returns an `onmessage` handler that re-arms `debounceTerminate()` (no reject — keep-alive TTL), returns the worker to the idle set, and resolves with `e.data`.
  - `private onErrorHandler(worker, reject)` — **L141-L153.** Returns an `onerror` handler that immediately terminates via `debounceTerminate(() => reject(e))` + `.flush()`, then calls `this.clear()` to drop any other idle workers. Side effect: an error in one worker flushes the whole pool.
