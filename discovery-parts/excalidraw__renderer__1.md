## Cluster: excalidraw__renderer__1

### packages/excalidraw/renderer/staticSvgScene.ts

Renders an Excalidraw scene into a live `SVGElement` DOM tree (the static SVG export path, distinct from the Canvas2D static-scene renderer), translating each element into rough.js-drawn SVG nodes, text `<text>` runs, `<image>`/`<use>` symbols, embeddable `<foreignObject>` iframes, and frame outlines/clip-paths.

This file has **4 module-level functions** (3 internal helpers + 1 exported entrypoint). `renderElementToSvg` is one large function with a `switch` over `element.type`; its per-case logic is documented below. No classes, hooks, or React components ‚Äî pure DOM/SVG construction. Imports are runtime symbols from `@excalidraw/common` and `@excalidraw/element` plus types from `../scene/types`, `../types`, and roughjs.

- **`roughSVGDrawWithPrecision(rsvg: RoughSVG, drawable: Drawable, precision?: number)` ‚Üí `SVGGElement`** ‚Äî L50-L64. Wraps `rsvg.draw()`. If `precision` is `undefined`, draws the drawable as-is. Otherwise it shallow-clones the drawable, injecting `options.fixedDecimalPlaceDigits = precision` so rough.js emits SVG path coordinates rounded to that many decimals. Used everywhere with `MAX_DECIMALS_FOR_SVG_EXPORT` to keep exported path data compact/deterministic. Side effect: none beyond creating the SVG node rough.js returns. Notable: it spreads `drawable.options` so it does not mutate the cached shape from `ShapeCache`.

- **`maybeWrapNodesInFrameClipPath(element: NonDeletedExcalidrawElement, root: SVGElement, nodes: SVGElement[], frameRendering: AppState["frameRendering"], elementsMap: RenderableElementsMap)` ‚Üí `SVGGElement | null`** ‚Äî L66-L85. If frame rendering is disabled OR frame `clip` is off, returns `null` (caller falls back to the raw node). Otherwise finds the element's containing frame via `getContainingFrame`; if found, creates a `<g>` with `clip-path="url(#<frame.id>)"` (set via `setAttributeNS(SVG_NS, ...)`), appends all `nodes` into it, and returns the `<g>`. Returns `null` when the element is not in a frame. Invariant: clip-path id matches the frame element id ‚Äî the `<clipPath>` definition itself is produced elsewhere (this only references it).

- **`renderElementToSvg(element, elementsMap, rsvg, svgRoot, files, offsetX, offsetY, renderConfig)` ‚Üí `void`** ‚Äî L87-L706. The core per-element dispatcher. Parameters: the element, the renderable map, the rough.js SVG instance, the root SVG node to append into, the `BinaryFiles` map (for image dataURLs), and translation offsets in SVG/world space plus `SVGRenderConfig`. Behavior detailed below by phase/case.

  - **Coordinate / rotation setup** ‚Äî L97-L117. Computes `[x1,y1,x2,y2] = getElementAbsoluteCoords(element)` (the rotated bounding box). Derives the rotation pivot `cx = (x2-x1)/2 - (element.x - x1)`, `cy = (y2-y1)/2 - (element.y - y1)` ‚Äî i.e. the element center expressed relative to the element's own origin, used as the `rotate()` center in SVG transforms. Converts `element.angle` (radians) to degrees: `degree = 180*angle/Math.PI`. **Coordinate-space note for parity:** every shape is positioned by an SVG `transform="translate(offsetX offsetY) rotate(degree cx cy)"`; the shape geometry itself is in element-local space, and translation+rotation are applied by the transform, never baked into coordinates.

  - **Bound-text-on-arrow special case** ‚Äî L101-L116. If the element is a text element whose container is an arrow, it recomputes `cx/cy` against the *container's* absolute coords and the bound-text position from `LinearElementEditor.getBoundTextElementPosition`, and shifts `offsetX/offsetY` by `boundTextCoords - element` so the label sits correctly along the arrow.

  - **Link anchor wrapping** ‚Äî L120-L128. If `element.link` is set, creates an `<a href=normalizeLink(link)>` appended to `svgRoot` and reassigns `root` to it, so the element renders inside the anchor.

  - **`addToRoot(node, element)`** ‚Äî L130-L135 (closure). Appends `node` to the current `root`; in test env (`isTestEnv()`) also stamps `data-id=element.id` for snapshot assertions.

  - **Opacity** ‚Äî L137-L140. `opacity = (containingFrame.opacity ?? 100) * element.opacity / 10000` ‚Äî composites the frame's opacity with the element's (each 0-100, product normalized to 0-1). Note the `embeddable` case (L189) overrides this with just `element.opacity/100`, ignoring frame opacity.

  - **case "selection"** ‚Äî L143-L147. Throws `"Selection rendering is not supported for SVG"` (canvas-only concept).

  - **case "rectangle" / "diamond" / "ellipse"** ‚Äî L148-L179. Generates the shape via `ShapeCache.generateElementShape`, draws with precision, sets `stroke-opacity`/`fill-opacity` when opacity‚âÝ1, `stroke-linecap=round`, and the translate+rotate transform. Optionally wraps in a frame clip-path `<g>` and appends.

  - **case "iframe" / "embeddable"** ‚Äî L180-L281. Renders a placeholder rect (rough shape) first, then a placeholder text label via a recursive `renderElementToSvg(createPlaceholderEmbeddableLabel(element), ...)`, then a second `embeddableNode` whose rough children are stripped (L229-L231) and replaced with either: (a) an `<a target=_blank rel=noopener>` link styled with `border-radius: radius` ‚Äî when `renderEmbeddables === false` or `embedLink.type === "document"` (srcdoc not supported in SVG); or (b) a `<foreignObject>` containing an XHTML `<div>` with an `<iframe src=embedLink.link allowFullscreen>` sized 100%. `radius = getCornerRadius(min(width,height), element)`. `embedLink = getEmbedLink(toValidURL(element.link||""))`. Opacity here = `element.opacity/100`.

  - **case "line" / "arrow"** ‚Äî L282-L376. If the element has bound text, builds an SVG `<mask id="mask-<id>">` with a white full-cover rect (`width = element.width+100+offsetX`, `height = element.height+100+offsetY`) and a black rect over the bound-text box (positioned at `offset + boundTextCoords - element`), so the line is "cut" behind its label. Wraps the rough shapes in a `<g stroke-linecap=round>` that references the mask. For closed `line` loops (`isPathALoop(element.points)`) with non-transparent background, sets `fill-rule=evenodd` (L351-L357). Per-shape opacity + transform as above. Frame-clip wrapping handled at L361-L374; note the branch appends both the clip `<g>` (with `addToRoot`) and re-appends it (`root.appendChild(g)`) on L369-L370, vs. appending `group` then the bare `maskPath` on L372-L373 ‚Äî parity-relevant DOM-ordering detail.

  - **case "freedraw"** ‚Äî L377-L436. Generates shapes ordered `[background(Drawable), stroke(SVGPathString)]`. For the string stroke it creates a `<path d=... fill=strokeColor>` (dark theme runs `applyDarkModeFilter` on the color). For the Drawable background it draws via rough; if rough returns a `<g>` it unwraps the children into the `wrapper` (L405-L408). Sets opacity, the translate+rotate transform, and `stroke="none"` on the wrapper (freedraw fills, not strokes). Optional frame clip-path wrap.

  - **case "image"** ‚Äî L437-L598. Rounds width/height to integers. Only renders if `isInitializedImageElement` and the file exists in `files`. **Symbol reuse / caching:** builds a `symbolId` ‚Äî default `image-<fileData.id>` (shared across reuse), `image-crop-<fileData.id>-<hashString("WxH")>` when cropped, or `image-<element.id>` when `reuseImages === false`. If no matching `<symbol>` exists, creates one containing an `<image href=dataURL preserveAspectRatio=none>` (sized to uncropped W/H when cropped or non-reuse, else `100%`), prepended into `<defs>` (or root). Then emits a `<use href=#symbolId>`. **Crop math** (L487-L497): `normalizedCropX = crop.x / (crop.naturalWidth / uncroppedWidth)` (same for Y) ‚Äî maps crop coords from natural-pixel space into element space; `adjustedCenter = (cx,cy) + normalizedCrop`. `<use>` width/height = `width/height + normalizedCrop`. **Mirroring** (L506-L521): when `element.scale[0|1] !== 1`, the `<use>` gets its own `translate(center) scale(sx sy) translate(-center)` transform ‚Äî done separately from the wrapping `<g>`'s translate+rotate to fix transform-origin compositing. Dark-mode SVG images get `filter=DARK_THEME_FILTER` (L525-L530). Crop applies a `<mask>` rect; `element.roundness` applies a `<clipPath>` with `rx=ry=getCornerRadius(min(w,h))`. Finally frame-clip-wrapped. **Performance/parity note:** the symbol-reuse cache means identical images share one `<symbol>` def referenced by multiple `<use>` nodes ‚Äî important to replicate to avoid bloated exports.

  - **case "frame" / "magicframe"** ‚Äî L599-L633. Frames are containers and normally invisible; only when `frameRendering.enabled && frameRendering.outline` does it emit a `<rect>` (rounded `rx=ry=FRAME_STYLE.radius`, `fill=none`, stroke=`FRAME_STYLE.strokeColor` dark-filtered, `stroke-width=FRAME_STYLE.strokeWidth`) with the translate+rotate transform.

  - **default ‚Üí text elements** ‚Äî L634-L704. For `isTextElement`, builds a `<g>` (opacity + transform) and one `<text>` per line (`element.text` split on normalized newlines). Layout math: `lineHeightPx = getLineHeightInPx(fontSize, lineHeight)`; per-line `y = i*lineHeightPx + verticalOffset` where `verticalOffset = getVerticalOffset(fontFamily, fontSize, lineHeightPx)` (baseline adjustment). `horizontalOffset` = `width/2` (center), `width` (right), or `0` (left). `text-anchor` = `middle`/`end`/`start` accordingly, with RTL (`isRTL(text)`) forcing `end`. Each `<text>` sets `font-family` (`getFontFamilyString`), `font-size`, `fill` (dark-filtered strokeColor), `direction`, `white-space: pre`, and `dominant-baseline=alphabetic`. Non-text unknown types throw `Unimplemented type ...` (L701, `@ts-ignore`).

- **`renderSceneToSvg(elements: readonly NonDeletedExcalidrawElement[], elementsMap: RenderableElementsMap, rsvg: RoughSVG, svgRoot: SVGElement, files: BinaryFiles, renderConfig: SVGRenderConfig)` ‚Üí `void`** ‚Äî L708-L786. The exported scene entrypoint. Guards on falsy `svgRoot`. **Two-pass render ordering:** pass 1 renders all non-iframe-like elements (filter `!isIframeLikeElement`), skipping deleted elements and text elements bound to a container present in `elementsMap` (those are drawn with their container); for each it calls `renderElementToSvg` at world position `element.x/y + renderConfig.offsetX/Y`, then renders any bound text element separately. Pass 2 renders iframe-like elements (`isIframeLikeElement`) **on top**, so embeddables/iframes always layer above other content. Errors per element are caught and `console.error`'d so one bad element doesn't abort the whole export. **Parity note:** the offset applied is `renderConfig.offsetX/Y` added to each element's absolute coordinates ‚Äî the export viewport translation is per-element, not a single root transform.
