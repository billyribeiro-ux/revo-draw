## Cluster: element__src__0

This cluster covers the geometry/binding core of the `@excalidraw/element` package: alignment, arrowhead normalization, arrow focus-point binding, element bounds computation, and collision/hit-testing. All geometry is in **world (scene) space** unless stated otherwise; rotation uses `pointRotateRads(point, center, angle)`.

---

### packages/element/src/align.ts

Purpose: aligns multiple selected elements (respecting groups) to a common axis edge/center.

- `interface Alignment` (L12-L15) — `{ position: "start" | "center" | "end"; axis: "x" | "y" }`.
- `alignElements(selectedElements, alignment, scene, appState): ExcalidrawElement[]` (L17-L50) — Splits the selection into groups via `getSelectedElementsByGroup`, computes the overall `selectionBoundingBox` (`getCommonBoundingBox`), then for each group computes a per-group `translation` and `flatMap`s the result. For each element it calls `scene.mutateElement` to shift `x`/`y` by the translation and `updateBoundElements(element, scene, { simultaneouslyUpdated: group })` so bound arrows follow. Side effects: mutates elements + bound elements. Invariant: a group moves as a rigid body (one translation per group).
- `calculateTranslation(group, selectionBoundingBox, { axis, position }): { x: number; y: number }` (L52-L80) — internal. Computes the group's own bounding box, picks `["minX","maxX"]` or `["minY","maxY"]` depending on axis. `start` aligns mins, `end` aligns maxes, `center` aligns midpoints; the off-axis component stays 0. Returns a delta object keyed by `[axis]`. Math: center delta = `(selMin+selMax)/2 - (grpMin+grpMax)/2`.

### packages/element/src/arrows/focus.ts

Purpose: manages the draggable arrow "focus point" indicator used to control where a (non-elbow) arrow binds on a bindable element (start/end), including orbit-vs-inside binding modes.

- `isFocusPointVisible(focusPoint, arrow, bindableElement, elementsMap, appState, startOrEnd, ignoreOverlap = false): boolean` (L36-L99) — Returns false for elbow arrows, when binding disabled, or when `arrow.points.length !== 2`. Unless `ignoreOverlap`, hides the indicator if the focus point sits within `(FOCUS_POINT_SIZE*1.5)/zoom` of its own associated arrow endpoint. Otherwise visible iff the focus point is far enough from the relevant arrow endpoint AND `hitElementItself` is true with `threshold = getBindingGap(...)` and `overrideShouldTestInside: true`. Coordinate note: hit thresholds are divided by `appState.zoom.value` (screen-constant size).
- `focusPointUpdate(arrow, bindableElement, isStartBinding, elementsMap, scene, appState, switchToInsideBinding)` (L102-L201) — internal. Recomputes arrow endpoint positions in "orbit"/"inside" configuration. Determines `bindingField`/`adjacentBindingField`; sets the current binding `mode` to `"inside"` if `switchToInsideBinding` or both ends bound to the same element, else `"orbit"`. Computes the new dragged endpoint via `updateBoundPoint(...)`. Also recomputes the adjacent endpoint if its binding is `"orbit"`. Collects updates into a `Map<pointIndex, {point}>` and applies them via `LinearElementEditor.movePoints` with updated binding objects. Side effect: mutates arrow points + bindings.
- `handleFocusPointDrag(linearElementEditor, elementsMap, pointerCoords, scene, appState, gridSize, switchToInsideBinding)` (L203-L332) — main drag handler. Sanity-checks (must be a binding element, non-elbow, with a dragged/hovered focus binding). Computes the dragged scene point minus `pointerOffset`. Uses `getHoveredElementForFocusPoint(point, arrow, scene elements, elementsMap, maxBindingDistance_simple(zoom))`. If hovering a bindable and binding enabled: breaks an existing binding to a different element, computes a mode switch (`"orbit"`↔`"inside"` via alt), calls `bindBindingElement` if no binding or mode change, then `scene.mutateElement(arrow, {...binding, fixedPoint via calculateFixedPointForNonElbowArrowBinding})`. If not hovering, moves the endpoint to the pointer (`LinearElementEditor.createPointAt`) and unbinds. Finally calls `focusPointUpdate` and, when hovering, `moveArrowAboveBindable` to fix z-order. Note: `arrow` is typed `as any` (L215).
- `handleFocusPointPointerDown(arrow, pointerDownState, elementsMap, appState): { hitFocusPoint, pointerOffset }` (L334-L423) — Hit-tests the start then end focus points (start takes precedence). For each, gets the global fixed point via `getGlobalFixedPointForBindableElement`, confirms `isFocusPointVisible`, and checks pointer distance ≤ `(FOCUS_POINT_SIZE*1.5)/zoom`. Returns which focus point was hit plus the `pointerOffset` (pointer minus focus point) for drag delta tracking, or `{ hitFocusPoint: null, pointerOffset: {0,0} }`.
- `handleFocusPointPointerUp(linearElementEditor, scene)` (L425-L486) — On release, reconciles `boundElements` bookkeeping. Asserts a dragged focus binding exists (`invariant`) and the arrow exists. Finds any old bindable element that still references the arrow in `boundElements` but is no longer the bound element, and removes the arrow from its `boundElements`. Then ensures the currently-bound element records the arrow (`{id, type:"arrow"}`) exactly once. Side effects: two possible `scene.mutateElement` calls.
- `handleFocusPointHover(arrow, scenePointerX, scenePointerY, scene, appState): "start" | "end" | null` (L488-L558) — Like pointer-down but read-only: returns `"start"`/`"end"` if the pointer is over a visible focus point (start precedence), else `null`. Threshold `(FOCUS_POINT_SIZE*1.5)/zoom`.

### packages/element/src/binding.ts

Purpose: the arrow↔shape binding engine — computes binding strategies for dragged arrow endpoints, binds/unbinds elements, recomputes bound arrow points when shapes move/resize/rotate, snaps endpoints to element outlines, and maintains bidirectional `boundElements` bookkeeping. Largest file in the cluster (2942 lines).

Constants (L112-L115): `BASE_BINDING_GAP = 5`, `BASE_BINDING_GAP_ELBOW = 5`, `BASE_ARROW_MIN_LENGTH = 10`, `FOCUS_POINT_SIZE = 10/1.5`. Gaps exclude strokeWidth and must be > 0.

Type `BindingStrategy` (L87-L105) — discriminated union: create binding `{mode: BindMode, element, focusPoint}`, break `{mode: null}`, or keep `{mode: undefined}`.

- `getBindingGap(bindTarget, opts): number` (L117-L125) — `(elbowed ? BASE_BINDING_GAP_ELBOW : BASE_BINDING_GAP) + strokeWidth/2`.
- `maxBindingDistance_simple(zoom?): number` (L127-L137) — Base distance `max(BASE_BINDING_GAP, 15)`; clamps `BASE/(zoomValue*1.5)` between `BASE` and `BASE*2`. zoomValue only applied when `< 1`. Reduces zoom impact at low zoom.
- `isBindingEnabled(appState): boolean` (L139-L143) — returns `appState.isBindingEnabled`.
- `bindOrUnbindBindingElement(arrow, draggingPoints, scenePointerX, scenePointerY, scene, appState, opts?)` (L145-L222) — Top-level finalize: computes `{start, end}` strategy (`finalize: true`), applies each edge via `bindOrUnbindBindingElementEdge`, and if either strategy has a `focusPoint`, recomputes endpoint(s) via `updateBoundPoint` and applies through `LinearElementEditor.movePoints`. Returns `{start, end}`.
- `bindOrUnbindBindingElementEdge(arrow, {mode,element,focusPoint}, startOrEnd, scene, shouldSnapToOutline = true)` (L224-L245) — internal. `mode === null` → unbind; `mode !== undefined` → `bindBindingElement`; `undefined` → no-op (keep).
- `bindingStrategyForElbowArrowEndpointDragging(arrow, draggingPoints, elementsMap, elements, zoom?)` (L247-L297) — internal. Asserts exactly one dragging point (`invariant`). Gets the global point, hovered element (`getHoveredElementForBinding`), and produces a `current` orbit strategy (focusPoint at the dragged endpoint) or `{mode: null}`; the other end stays `undefined`.
- `bindingStrategyForNewSimpleArrowEndpointDragging(...)` (L299-L455) — internal, complex-binding path for newly-created arrows. Handles start-drag (bind inside if hit else break), and end-drag covering: inside→inside (same element), nested-shape detection via `getAllHoveredElementAtPoint`, inside→outside transitions (orbit vs inside based on `arrowStartIsInside`/`globalBindMode`), and no-start-binding cases. Uses `appState.selectedLinearElement.initialState.origin` as focus anchor. Ends with `invariant(false, ...)` guarding unreachable states.
- `bindingStrategyForSimpleArrowEndpointDragging_complex(point, currentBinding, oppositeBinding, elementsMap, elements, globalBindMode, arrow, finalize?)` (L457-L579) — internal. Computes `{current, other}` strategies for an existing arrow's dragged endpoint. Handles free `"inside"`/`"skip"` modes, overlapping/transparent opposite element detection, nested-bindable detection, and orbit-vs-inside selection. For multi-point arrows (`points.length > 2`) it returns `other: {mode: undefined}` to avoid clobbering the opposite end.
- `getBindingStrategyForDraggingBindingElementEndpoints(arrow, draggingPoints, screenPointerX, screenPointerY, elementsMap, elements, appState, opts?)` (L581-L619) — public dispatcher: routes to `_complex` if feature flag `COMPLEX_BINDINGS` is on, else `_simple`.
- `getBindingStrategyForDraggingBindingElementEndpoints_simple(...)` (L621-L869) — internal. The default binding-strategy resolver. Early-outs: nothing dragged → keep; both dragged → break both; binding disabled → break dragged end. Elbow arrows delegate. Computes hovered element with `maxBindingDistance_simple(zoom)`, `pointInElement` test (uses `scenePointer` when angle-locked), and the opposite binding's focus point/in-element status. Handles outside-outside binding to the same element (both `inside`), alt-key force-inside, and the normal orbit/inside decision using `projectFixedPointOntoDiagonal` for orbit focus points. Returns `{start, end}` mapping `current`/`other` per which end was dragged.
- `getBindingStrategyForDraggingBindingElementEndpoints_complex(...)` (L871-L997) — internal. Complex-mode variant using `appState.bindMode` (default `"orbit"`). Same early-outs; routes new arrows to `bindingStrategyForNewSimpleArrowEndpointDragging`; otherwise calls `bindingStrategyForSimpleArrowEndpointDragging_complex` for start or end.
- `bindOrUnbindBindingElements(selectedArrows, scene, appState)` (L999-L1014) — Re-finalizes bindings for each arrow with empty dragging points and pointer at `Infinity`.
- `bindBindingElement(arrow, hoveredElement, mode, startOrEnd, scene, focusPoint?, shouldSnapToOutline = true)` (L1016-L1068) — Builds a `FixedPointBinding`: elbow arrows always `"orbit"` with `calculateFixedPointForElbowArrowBinding`; others use given `mode` + `calculateFixedPointForNonElbowArrowBinding`. Sets `startBinding`/`endBinding` on the arrow and adds `{id, type:"arrow"}` to the target's `boundElements` (dedup via `arrayToMap`). Side effects: 1-2 `scene.mutateElement`.
- `unbindBindingElement(arrow, startOrEnd, scene): id | null` (L1070-L1100) — Clears the chosen binding. Only removes the arrow from the target's `boundElements` if the opposite end isn't bound to the same element. Returns the previously-bound element id, or null.
- `updateBoundElements(changedElement, scene, options?)` (L1102-L1204) — Core "shape moved → fix arrows" routine. No-op unless `changedElement` is bindable. Optionally overlays `options.changedElements` onto a copied elementsMap. For each bound arrow (via `boundElementsVisitor`): skips deleted/stale/`simultaneouslyUpdated` arrows, finds start/end bindable elements (with a PERF shortcut when both ends share an element), recomputes affected endpoint points via `bindableElementsVisitor` + `updateBoundPoint`, applies with `LinearElementEditor.movePoints` (sets `moveMidPointsWithElement` when both ends bound to same element), and resizes any bound text via `handleBindTextResize`.
- `updateArrowBindings(latestElement, startOrEnd, elementsMap, scene, appState)` (L1206-L1266) — internal. For a moved arrow (not the shape): asserts non-elbow, unbinds the end, then if the endpoint still hits its bindable (`hitElementItself` within `maxBindingDistance_simple`), recomputes a strategy and re-binds.
- `updateBindings(latestElement, scene, appState, options?)` (L1268-L1305) — If the changed element is an arrow, calls `updateArrowBindings` for each present binding; otherwise calls `updateBoundElements` with `changedElements` seeded by the element itself.
- `doesNeedUpdate(boundElement, changedElement): boolean` (L1307-L1315) — true if either binding references the changed element id.
- `getSimultaneouslyUpdatedElementIds(simultaneouslyUpdated): Set<id>` (L1317-L1321) — internal Set builder.
- `getHeadingForElbowArrowSnap(p, otherPoint, bindableElement, aabb, origPoint, elementsMap, zoom?): Heading` (L1323-L1352) — Picks an elbow-arrow snap heading: if no element/aabb, heading from `otherPoint→p`; if `origPoint` is beyond binding distance, heading from `p→center`; otherwise `headingForPointFromElement`.
- `getDistanceForBinding(point, bindableElement, elementsMap, zoom?)` (L1354-L1364) — internal. Returns `distanceToElement` if ≤ `maxBindingDistance_simple`, else null.
- `bindPointToSnapToElementOutline(arrowElement, bindableElement, startOrEnd, elementsMap, customIntersector?, isMidpointSnappingEnabled = true): GlobalPoint` (L1366-L1502) — Heavy geometry: snaps an arrow endpoint onto a shape's outline. For elbow arrows it avoids rectangular corners (`avoidRectangularCorner`), optionally snaps to mid (`snapToMid`), constructs a directional `lineSegment` from the shape center, and finds the nearest intersection (`intersectElementWithLineSegment(...).sort(pointDistanceSq)[0]`), with a perpendicular fallback. Returns the intersection, or `edgePoint` if none / too close (`< PRECISION`).
- `avoidRectangularCorner(arrowElement, bindTarget, elementsMap, p): GlobalPoint` (L1504-L1593) — De-rotates `p` about the target center, classifies which corner quadrant it's in, and nudges it onto the nearer adjacent edge offset by `bindingGap`, then re-rotates. Prevents elbow arrows from snapping to a corner.
- `snapToMid(bindTarget, elementsMap, p, tolerance = 0.05, arrowElement?): GlobalPoint | undefined` (L1595-L1708) — De-rotates `p`, and if within an adaptive threshold band (`clamp(tolerance*dim, 5, 80)` px) of a side's midline, returns the rotated side-midpoint offset by `bindingGap`. Returns undefined if too close to center (`< bindingGap`). For diamonds, additionally snaps to the four quarter-vertices. Note: center uses `elementCenterPoint(..., -0.1, -0.1)`.
- `extractBinding(arrow, startOrEnd, elementsMap)` (L1710-L1741) — internal. Resolves a binding into `{element, fixedPoint, focusPoint (global), binding, mode}`, or all-null if no binding.
- `elementArea(element)` (L1743-L1744) — `width * height`.
- `updateBoundPoint(arrow, startOrEnd, binding, bindableElement, elementsMap, dragging?): LocalPoint | null` (L1746-L1902) — Computes the new local arrow endpoint for a binding. Returns null when binding is null, when the binding is on a different element for a >2-point arrow, or for the initial (0,0) arrow. Short-circuits `"inside"` bindings to the focus point. Otherwise builds an intersector segment between this focus point and the opposite focus/arrow point, finds outline intersections on both bindable elements, and decides among: (1) other-shape-contains-outline → focus point (anti-inversion), (2) unconnected other end → outline unless too short, (3) too-short both-ends → resolved target, (4) general → outline point. - `fixDuplicatedBindingsAfterDuplication(duplicatedElements, origIdToDuplicateId, duplicateElementsMap)` (L1989-L2059) — Remaps `boundElements`, `containerId`, `startBinding.elementId`, `endBinding.elementId` from original ids to duplicate ids (dropping bindings whose target wasn't duplicated). ### packages/element/src/bounds.ts

### packages/element/src/collision.ts

Purpose: precise hit-testing and line-segment↔element intersection for shapes (rectangle/diamond/ellipse/linear/freedraw), plus binding-border tests and point-in-element/outline tests. World-space throughout; rotated shapes are tested by inverse-rotating the query.

- `shouldTestInside(element): boolean` (L82-L102) — Whether a shape should be hit-tested by its interior. Arrows never. "Draggable from inside" requires opaque background, bound text, iframe-like, or text; lines/freedraw additionally require a closed loop (`isPathALoop`); images always count.
- `type HitTestArgs` (L104-L111) — `{point, element, threshold, elementsMap, frameNameBound?, overrideShouldTestInside?}`.
- Module-level hit cache (L113-L117): `cachedPoint`, `cachedElement` (`WeakRef`), `cachedThreshold`, `cachedHit`, `cachedOverrideShouldTestInside`.
- `hitElementItself(HitTestArgs): boolean` (L119-L192) — Main precise hit test. Returns cached result when the same point + element (id/version/versionNonce) + override flag is retested and threshold ≤ cached. Otherwise: optional frame-name bbox test; cheap rotated-bbox pre-check (`isPointInRotatedBounds`) that bails 99% of the time; then `shouldTestInside`/override decides whether to combine `isPointInElement` with `isPointOnElementOutline` or just the outline. Caches and returns. PERF: the early bbox bailout is the key optimization.
- `isPointInRotatedBounds(point, bounds, angle, tolerance = 0): boolean` (L194-L210) — internal. Inverse-rotates the point about the bounds center (when angle≠0) and tests against tolerance-expanded bounds.
- `hitElementBoundingBox(point, element, elementsMap, tolerance = 0)` (L212-L220) — Rotated-bbox-only hit (uses non-rotated bounds + element angle).
- `hitElementBoundingBoxOnly(hitArgs, elementsMap)` (L222-L229) — true when the point hits the bbox but neither the element itself nor its bound text.
- `hitElementBoundText(point, element, elementsMap): boolean` (L231-L255) — Tests the point against a bound text element; for linear containers it uses the computed `getBoundTextElementPosition`.
- `bindingBorderTest(element, [x,y], elementsMap, tolerance = 0): boolean` (L257-L303) — Whether a point is close enough to bind to `element`. Cheap AABB pre-check (`t = max(1,tolerance)`), frame-clipping check (must be inside the enclosing frame), then intersects a center→point segment and measures `distanceToElement`. Frames disable full-shape snapping (so children are bindable); shaped elements need `intersections===0 || distance<=tolerance` (inside), frames need `intersections>0 && distance<=t` (border).
- `getAllHoveredElementAtPoint(point, elements, elementsMap, tolerance?): bindable[]` (L305-L339) — Iterates elements front-to-back (high z-index first), collecting bindable elements passing `bindingBorderTest`; stops at the first opaque-background element. Asserts no deleted elements (`invariant`).
- `getHoveredElementForBinding(point, elements, elementsMap, tolerance?): bindable | null` (L341-L368) — Returns the single best binding candidate; when multiple, prefers the **smaller** shape (sorts by `w²+h²` descending and pops).
- `getHoveredElementForFocusPoint(point, arrow, elements, elementsMap, tolerance?): bindable | null` (L370-L418) — Like above but for focus-point dragging; filters candidates by `distanceToElement <= getBindingGap` or point-inside, returns the first.
- `intersectElementWithLineSegment(element, elementsMap, line, offset = 0, onlyFirst = false): GlobalPoint[]` (L428-L491) — Dispatcher: cheap AABB pre-check (offset-expanded segment vs element bounds), then per-type intersection (rectanguloid/diamond/ellipse/linear-or-freedraw). Returns intersection points in world space.
- `curveIntersections(curves, segment, intersections, center, angle, onlyFirst)` (L493-L529) — internal. Per Bézier curve: cheap bbox check (`getCubicBezierCurveBound` vs segment bbox), then `curveIntersectLineSegment`, rotating hits back by `angle`. Supports early return.
- `lineIntersections(lines, segment, intersections, center, angle, onlyFirst)` (L531-L551) — internal. Per side: `lineSegmentIntersectionPoints`, rotating hits back by `angle`.
- `intersectLinearOrFreeDrawWithLineSegment(element, segment, elementsMap, onlyFirst)` (L553-L606) — internal. Uses `deconstructLinearOrFreeDrawElement` (which returns already-rotated lines/curves), tests lines then curves (with bbox pre-check and `iterLimit: 10`); does **not** re-rotate.
- `intersectRectanguloidWithLineSegment(element, elementsMap, segment, offset, onlyFirst)` (L608-L658) — internal. Inverse-rotates the segment about center, deconstructs into sides+corner curves (offset), tests sides then corners.
- `intersectDiamondWithLineSegment(element, elementsMap, l, offset, onlyFirst)` (L667-L708) — internal. Same pattern as rectanguloid for diamonds.
- `intersectEllipseWithLineSegment(element, elementsMap, l, offset)` (L717-L732) — internal. Inverse-rotates segment, calls `ellipseSegmentInterceptPoints` on an `ellipse(center, w/2+offset, h/2+offset)`, re-rotates hits.
- `isPointOnElementOutline(point, element, elementsMap, tolerance = 1): boolean` (L742-L747) — internal. `distanceToElement <= tolerance`.
- `isPointInElement(point, element, elementsMap): boolean` (L756-L791) — Ray-casting point-in-shape: non-looping linear/freedraw have no inside (false); bbox pre-check; casts a ray from the point to a far point through the center (segment), counts unique intersections (`intersectElementWithLineSegment` deduped via `pointsEqual`), and returns odd-parity. Classic even-odd rule.
- `isBindableElementInsideOtherBindable(innerElement, outerElement, elementsMap): boolean` (L793-L849) — Whether the inner shape is fully contained in the outer. Computes the inner shape's representative corner/extreme points per type (diamond mid-edges, ellipse 4 extremes, rectangle 4 corners), rotated about the inner center, expanded by a **negative 5% offset** (`-max(w,h)/20`, shrinking inward), and tests every point is `isPointInElement` of the outer.

Coordinate/geometry notes for parity: all rotated-shape intersection tests inverse-rotate the query segment/point rather than rotating the shape; the `hitElementItself` single-entry WeakRef cache and the AABB pre-checks are the main performance levers; point-in-element uses the even-odd ray-casting rule with a ray length of `max(width,height)*2` from the center.
