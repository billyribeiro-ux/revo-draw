## Cluster: excalidraw__lasso

This cluster implements Excalidraw's **lasso selection tool**: a freehand-drawn path (animated trail) that selects elements either by *enclosing* them or by *intersecting* their geometry. The cluster has two files — `lasso/index.ts` (the `LassoTrail` controller class) and `lasso/utils.ts` (the pure geometry/selection-test functions). All coordinates are in **world/scene space** (`GlobalPoint`); the lasso path comes from the underlying `AnimatedTrail`'s captured points and is compared against per-element line segments and bounds.

---

### packages/excalidraw/lasso/index.ts

Purpose: Defines `LassoTrail`, a subclass of `AnimatedTrail` that captures a freehand drag, renders it as a translucent purple animated stroke, and continuously recomputes the selected element set (handling text-container promotion, frame-child de-duplication, group selection, and single-linear-element editor activation).

Top-of-file declarations:
- `type CanvasTranslate = { scrollX: number; scrollY: number; zoom: number }` (L34-L38) — snapshot of the camera/viewport state, used to cache element segment computations and invalidate that cache when the view pans/zooms.

Class `LassoTrail extends AnimatedTrail` (L40-L218). Private instance fields (L41-L46):
- `intersectedElements: Set<ExcalidrawElement["id"]>` — ids whose geometry the lasso path crosses (mutated in `utils.ts`).
- `enclosedElements: Set<ExcalidrawElement["id"]>` — ids fully inside the lasso polygon (mutated in `utils.ts`).
- `elementsSegments: Map<string, LineSegment<GlobalPoint>[]> | null` — cache of each visible element's line segments, keyed by element id; `null` until first computed/after `endPath`.
- `canvasTranslate: CanvasTranslate | null` — last camera state for which `elementsSegments` was built; used to detect when to rebuild the cache.
- `keepPreviousSelection: boolean` — when true (e.g. additive/modifier drag), merges the lasso result with the pre-existing selection instead of replacing it.

Methods:

- **`constructor(app: App)`** (L48-L69) — Calls `super(app, {...})` with the `AnimatedTrail` config that defines the lasso's visual identity. `animateTrail: true`; `streamline: 0.4` (smoothing factor for the freehand curve). `sizeMapping(c)` computes per-point trail thickness via two decay factors: a time decay `t = max(0, 1 - (now - c.pressure)/DECAY_TIME)` where `DECAY_TIME = Infinity` (so `t` is effectively always 1), and a length decay `l = (DECAY_LENGTH - min(DECAY_LENGTH, c.totalLength - c.currentIndex)) / DECAY_LENGTH` with `DECAY_LENGTH = 5000`; returns `min(easeOut(l), easeOut(t))`. Fill is `rgba(105,101,219,0.05)` (faint interior tint), stroke is `rgba(105,101,219)` (solid purple). Notable: because `DECAY_TIME = Infinity`, the time term degenerates to 1 and the visible taper is driven entirely by the length term — points within the last 5000 units of trail length fade toward the tip.

- **`startPath(x: number, y: number, keepPreviousSelection = false)`** (L71-L88) — Begins a new lasso gesture. First defensively calls `this.endPath()` to clear any stray trail, then `super.startPath(x, y)`. Clears `intersectedElements` and `enclosedElements`, records `keepPreviousSelection`. If NOT keeping previous selection, immediately resets app state (`selectedElementIds: {}`, `selectedGroupIds: {}`, `selectedLinearElement: null`). Side effect: mutates app state via `this.app.setState`.

- **`selectElementsFromIds = (ids: string[])`** (L90-L162) — Arrow-function field that translates a raw id list into the canonical selection state through `app.setState((prevState) => {...})`. Algorithm: (1) build `nextSelectedElementIds` map from `ids` (L92-L95); (2) if `keepPreviousSelection`, OR-in all ids from `prevState.selectedElementIds` (L97-L101); (3) **text-container promotion** — for each selected text element that has a container, select the container and delete the text element id (L103-L116); (4) **frame-child de-dup** — for each selected frame-like element, remove all of its children from the selection so the frame represents them (L118-L131); (5) run `selectGroupsForSelectedElements(...)` to expand to full group selection (L133-L142); (6) derive `selectedIds`/`selectedGroupIds` arrays (L143-L144); (7) return new state, and if exactly one element is selected, no groups, and that element is linear, instantiate a `LinearElementEditor` for it, else `selectedLinearElement: null` (L146-L160). Invariants: text inside a container is never selected independently of its container; frame children are subsumed by their frame.

- **`addPointToPath = (x: number, y: number, keepPreviousSelection = false)`** (L164-L170) — Arrow-function field. Calls `super.addPointToPath(x, y)` to extend the trail, updates `keepPreviousSelection`, then triggers `this.updateSelection()`. This is the per-pointer-move hot path.

- **`private updateSelection = ()`** (L172-L209) — Arrow-function field; the core recompute. (1) Reads the current trail's `originalPoints` and maps them to `GlobalPoint[]` via `pointFrom` (L173-L175). (2) Builds `currentCanvasTranslate` from `scrollX/scrollY/zoom` (L177-L181). (3) **Segment cache invalidation**: if `elementsSegments` is null OR the camera changed (`!isShallowEqual(currentCanvasTranslate, this.canvasTranslate ?? {})`), rebuild the cache by computing `getElementLineSegments(element, visibleElementsMap)` for every visible element (L183-L194). (4) If a `lassoPath` exists, call `getLassoSelectedElementIds({...})` with `simplifyDistance: 5 / zoom` (so the path simplification tolerance is constant in *screen* pixels regardless of zoom), then apply the result via `selectElementsFromIds` (L196-L208). Performance detail: the per-element segment cache means segments are only recomputed when the viewport transform changes, not on every pointer move; `intersectedElements`/`enclosedElements` Sets are passed in and reused.

- **`endPath(): void`** (L211-L217) — Ends the gesture: `super.endPath()`, `super.clearTrails()`, clears both id Sets, and nulls out `elementsSegments` so the next gesture rebuilds the cache. Side effect: releases the cached segment map.

Coordinate-space note: everything operates in world space (`GlobalPoint`); the lasso path points come straight from `AnimatedTrail.originalPoints`. The only zoom-dependent quantity is the path-simplification tolerance (L204), which is intentionally scaled by `1/zoom` to keep it perceptually constant.

### packages/excalidraw/lasso/utils.ts

Purpose: Pure helper module computing which elements a lasso path selects, via an AABB broad-phase filter followed by an enclosure test (polygon-contains-point) and an intersection test (segment-vs-element-shape).

- **`getLassoSelectedElementIds(input)`** (L22-L90) — Exported. Signature: `(input: { lassoPath: GlobalPoint[]; elements: readonly ExcalidrawElement[]; elementsMap: ElementsMap; elementsSegments: ElementsSegmentsMap; intersectedElements: Set<...id>; enclosedElements: Set<...id>; simplifyDistance?: number }) => { selectedElementIds: string[] }`. Algorithm: (1) optionally simplify the path with `simplify(lassoPath, simplifyDistance)` from `points-on-curve` to cut point count (L42-L46); (2) filter out locked elements (`unlockedElements`, L47); (3) clear both `enclosedElements` and `intersectedElements` Sets up front, because as the path changes a previously-enclosed shape may no longer be enclosed (L48-L50); (4) compute `lassoBounds` as an AABB `[minX, minY, maxX, maxY]` by reducing over the *original unsimplified* `lassoPath` (L51-L61); (5) for each unlocked element, broad-phase test `doBoundsIntersect(lassoBounds, getElementBounds(element, elementsMap))` (and skip if already classified) — only if the AABBs overlap does it run the expensive shape tests (L62-L82); (6) run `enclosureTest` first; if enclosed, add to `enclosedElements`, otherwise run `intersectionTest` and add to `intersectedElements` on a hit (L73-L81); (7) return `selectedElementIds = [...intersectedElements, ...enclosedElements]` (L85-L89). Side effects: mutates the passed-in `intersectedElements`/`enclosedElements` Sets (these are the caller's caches). Performance: the AABB broad-phase (`doBoundsIntersect`) is explicitly chosen because it is far cheaper than per-shape geometry tests (comment L63-L65). Subtle detail: bounds are computed from `lassoPath` (raw) while enclosure/intersection use `path` (possibly simplified).

- **`enclosureTest(lassoPath, element, elementsSegments): boolean`** (L92-L108) — Internal. Builds a polygon from the lasso path via `polygonFromPoints(lassoPath)`, fetches the element's precomputed `segments` from `elementsSegments`; returns `false` if no segments exist. Returns true if *any* point of *any* of the element's line segments lies inside the lasso polygon, tested with `polygonIncludesPointNonZero(point, lassoPolygon)` (non-zero winding-number containment rule). Note: this treats an element as "enclosed" if even one of its segment endpoints is inside the polygon — it is a containment heuristic over segment vertices, not a strict full-shape-inside test.

- **`intersectionTest(lassoPath, element, elementsMap): boolean`** (L110-L143) — Internal. Converts the lasso path into a closed sequence of `lineSegment`s: pairs each point with its predecessor (`lassoPath.slice(1).map(...lineSegment(lassoPath[index], point))`) and appends a closing segment from the last point back to the first (L115-L118). Fetches any bound text element via `getBoundTextElement`. Returns true if *any* lasso segment intersects the element's shape (`intersectElementWithLineSegment(element, elementsMap, lassoSegment, 0, true).length > 0`) OR intersects the element's bound text element (positioned via `computeBoundTextPosition`, spread onto the bound text) (L122-L142). The `0` arg is the intersection tolerance/offset and `true` enables the relevant intersection mode flag of `intersectElementWithLineSegment`. Geometry detail: the lasso polygon is explicitly closed by the appended final segment so a shape edge crossing the "open mouth" of the freehand stroke is still detected.

Imports / dependencies of note: `simplify` from `points-on-curve` (path decimation); `polygonFromPoints`, `lineSegment`, `polygonIncludesPointNonZero` from `@excalidraw/math`; `computeBoundTextPosition`, `doBoundsIntersect`, `getBoundTextElement`, `getElementBounds`, `intersectElementWithLineSegment` from `@excalidraw/element`. Types `ElementsSegmentsMap`/`GlobalPoint` (math) and `ElementsMap`/`ExcalidrawElement` (element).
