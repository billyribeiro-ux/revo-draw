### packages/excalidraw/data/index.ts

Top-level data-module barrel that prepares elements for export and drives canvas/SVG/PNG/clipboard export.

- `ExportedElements` (type) — `readonly NonDeletedExcalidrawElement[] & { _brand: "exportedElements" }`. Branded type marking an array that has already been filtered/cloned for export. (L43-L45)
- `prepareElementsForExport(elements: readonly ExcalidrawElement[], { selectedElementIds }: Pick<AppState, "selectedElementIds">, exportSelectionOnly: boolean)` — returns `{ exportingFrame: ExcalidrawFrameLikeElement | null; exportedElements: ExportedElements }`. Strips deleted elements, decides whether only the current selection is being exported, and resolves three cases: a single selected frame (exports all elements overlapping the frame), multiple selected elements (includes bound text + in-frame elements), or the whole scene. Returns a deep `cloneJSON` of the chosen elements so downstream mutation is safe. Invariant: output elements are always non-deleted and cloned (not references into the live scene). (L47-L97)
- `exportCanvas(type: Omit<ExportType, "backend">, elements: ExportedElements, appState: AppState, files: BinaryFiles, opts: { exportBackground; exportPadding=DEFAULT_EXPORT_PADDING; viewBackgroundColor; name=appState.name||DEFAULT_FILENAME; fileHandle=null; exportingFrame: ExcalidrawFrameLikeElement|null })` — async. The export dispatcher. Throws `t("alerts.cannotExportEmptyCanvas")` on empty input. For `"svg"`/`"clipboard-svg"` it calls `exportToSvg` (passing `exportEmbedScene` only when type==="svg"); for `"svg"` it saves a Blob prefixed with `SVG_DOCUMENT_PREAMBLE`+`svg.outerHTML`, for `"clipboard-svg"` it copies SVG text to clipboard. Otherwise renders via `exportToCanvas`: `"png"` saves a Blob, lazily importing `./image`'s `encodePngMetadata` to embed the serialized scene when `exportEmbedScene`; `"clipboard"` copies the PNG blob to the system clipboard with Firefox/CANVAS_POSSIBLY_TOO_BIG specific error messaging. Falls through to `throw new Error("Unsupported export type")`. Side effects: writes files via `fileSave`, writes to system clipboard, dynamic import of the image-metadata encoder. Non-obvious: filename extension becomes `excalidraw.svg`/`excalidraw.png` when embedding the scene so the file is re-importable. (L99-L220)
- Re-exports: `loadFromBlob` from `./blob`; `loadFromJSON`, `saveAsJSON` from `./json`. (L40-L41)
