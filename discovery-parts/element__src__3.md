### packages/element/src/groups.ts

Group selection logic: computing which group IDs to select when elements are selected, mapping groups↔elements, and group membership predicates. Heavily memoized for the interactive canvas hot path.

- **`selectGroup(groupId, appState, elements)` → `Pick<InteractiveCanvasAppState, "selectedGroupIds" | "selectedElementIds" | "editingGroupId">`** (L23-L63). Builds a `Record<id,true>` of all elements whose `groupIds` include `groupId`. If fewer than 2 members, the group is not a real group: if it was selected/editing it returns it cleared (`[groupId]: false`, `editingGroupId: null`), else returns appState unchanged. Otherwise returns the group marked selected plus all member element IDs merged into `selectedElementIds`. Pure.
- **`selectGroupsForSelectedElements` IIFE** (L65-L208). Closure with memo cache fields `lastSelectedElements`, `lastElements`, `lastReturnValue`.
  - **inner `_selectGroups(selectedElements, elements, appState, prevAppState)` → `SelectGroupsReturnType`** (L76-L158). Memo short-circuit when `elements`, `selectedElements`, and `editingGroupId` are reference-equal to last call (L82-L89). Otherwise: for each selected element, take the *last* (outermost-but-respecting-edit) group id — if `editingGroupId` is in the element's `groupIds`, slice the array before that index so nested editing only selects the inner group (L93-L106). Then walks all non-deleted elements building `selectedElementIdsInGroups` and a `groupElementsIndex` (groupId→element ids) (L109-L131). Singleton groups (`< 2` members) get unselected (L133-L140). Result wraps `selectedElementIds` through `makeNextSelectedElementIds` for referential-equality preservation (L148-L154). Caches and returns.
  - **outer `selectGroupsForSelectedElements(appState, elements, prevAppState, app)`** (L164-L199). Resolves selected elements via `app.scene.getSelectedElements` (preferred, optimized) or fallback `getSelectedElements`. Empty → returns cleared groups/editing + `makeNextSelectedElementIds`. Otherwise delegates to `_selectGroups`.
  - **`.clearCache()`** (L201-L205) nulls the three memo fields. Invariant: cache must be cleared when scene mutates outside reference tracking.
- **`isSelectedViaGroup(appState, element)`** (L214-L217). True if `getSelectedGroupForElement` returns non-null — used to suppress per-element selection borders when its group is selected.
- **`getSelectedGroupForElement(appState, element)`** (L219-L228). Filters out the `editingGroupId`, then finds the first of the element's `groupIds` present in `selectedGroupIds`.
- **`getSelectedGroupIds(appState)` → `GroupId[]`** (L230-L235). Entries of `selectedGroupIds` filtered to `isSelected===true`.
- **`selectGroupsFromGivenElements(elements, appState)`** (L239-L266). Reduces over elements, taking each element's outermost group (slicing before `editingGroupId`), and folds `selectGroup` results into a running `nextAppState`. Returns the resulting `selectedGroupIds`.
- **`editGroupForSelectedElement(appState, element)` → `AppState`** (L268-L280). Enters edit mode for the element's first (innermost) group: sets `editingGroupId` to `groupIds[0]`, clears `selectedGroupIds`, selects only that element.
- **`isElementInGroup(element, groupId)`** (L282-L283). `groupIds.includes`.
- **`getElementsInGroup(elements, groupId)`** (L285-L296). Iterates `elements.values()` (works on Map or array), collects members.
- **`getSelectedGroupIdForElement(element, selectedGroupIds)`** (L298-L301). First of element's `groupIds` that is selected.
- **`addToGroup(prevGroupIds, newGroupId, editingGroupId)`** (L303-L317). Inserts `newGroupId` *before* `editingGroupId` (so the new group nests outside the editing group) or pushes to end. Invariant: group order = inner→outer.
- **`removeFromSelectedGroups(groupIds, selectedGroupIds)`** (L319-L322). Filters out any selected group ids.
- **`getMaximumGroups(elements, elementsMap)` → `ExcalidrawElement[][]`** (L324-L349). Buckets elements by their outermost group id (or own id if ungrouped); each bucket also includes the element's bound-text element (via `getBoundTextElement`). Returns array of "maximal groups". Note: ungrouped elements become singleton groups keyed by id; `Map<String,...>` uses boxed `String` keys (cosmetic).
- **`getNonDeletedGroupIds(elements)` → `Set<string>`** (L351-L367). Collects all group ids across non-deleted elements; defensive `?? []` fallback on `groupIds`.
- **`elementsAreInSameGroup(elements)`** (L369-L384). Flattens all `groupIds`, counts occurrences; true iff some single group id occurs in *every* element (`maxGroup === elements.length`).
- **`isInGroup(element)`** (L386-L388). `groupIds.length > 0`.
- **`getNewGroupIdsForDuplication(groupIds, editingGroupId, mapper)`** (L390-L406). Remaps group ids for the indices *before* `editingGroupId` (or all if not editing) via `mapper` — so duplication generates fresh inner group ids while keeping the outer editing group shared.
- **`getSelectedElementsByGroup(selectedElements, elementsMap, appState)` → `ExcalidrawElement[][]`** (L410-L481). Buckets selected (non-bound-to-container) elements into per-group arrays and per-element arrays. Helpers `addToElementsMap` (L423-L432) and `addToGroupsMap` (L435-L444) both append bound text. `handleSingleSelectedGroupCase` (L449-L461) respects nested grouping order when exactly one group is selected and all elements are in it: if the element has groups nested inside the selected one, it buckets by the next-inner group id, else treats as individual. Returns `groups.values()` concatenated with `elements.values()`.

### packages/element/src/heading.ts

Computes a cardinal "heading" (one of 4 unit vectors) from a point relative to a bindable element's bounding box — core geometry for elbow-arrow routing and binding focus. No coordinate-space conversion (operates in world/global space).

- **Constants** `HEADING_RIGHT [1,0]`, `HEADING_DOWN [0,1]`, `HEADING_LEFT [-1,0]`, `HEADING_UP [0,-1]` and `type Heading` union (L31-L35). NOTE: y-down screen convention — DOWN is `[0,1]`.
- **`vectorToHeading(vec)` → `Heading`** (L37-L49). Snaps an arbitrary vector to the nearest of 4 cardinal directions by comparing signed components against `|x|`/`|y|`: `x > |y|`→RIGHT, `x <= -|y|`→LEFT, `y > |x|`→DOWN, else UP. Quadrant decision via abs comparison (diagonal ties resolve UP/LEFT).
- **`headingForPoint<P>(p, o)`** (L51-L54). `vectorToHeading(vectorFromPoint(p, o))` — heading of `p` as seen from origin `o`.
- **`headingForPointIsHorizontal<P>(p, o)`** (L56-L59). Convenience: heading is horizontal.
- **`compareHeading(a, b)`** (L61-L62). Component equality.
- **`headingIsHorizontal(a)`** (L64-L65). Equals RIGHT or LEFT.
- **`headingIsVertical(a)`** (L67). Negation of horizontal.
- **`headingForPointFromDiamondElement(element, aabb, point)` → `Heading`** (internal, L69-L226). Diamond-specific heading. Computes midpoint via `getCenterForBounds`. In dev/test, invariants assert non-zero size and that `point` ≠ midpoint (L76-L85). Builds the 4 diamond vertex midpoints (top/right/bottom/left), each rotated by `element.angle` about midpoint then scaled toward midpoint by `SHRINK=0.95` (rounded-corner tolerance) (L87-L149). Uses **vector cross products** to test which "corner" or "side" sector `point` falls into; for side sectors it picks top/right/bottom/left depending on whether the diamond is wider than tall (`element.width > element.height`) (L151-L225). Heavy 2D geometry — important for parity.
- **`headingForPointFromElement<Point>(element, aabb, p)` → `Heading`** (L231-L281). General (non-diamond) case. `SEARCH_CONE_MULTIPLIER = 2`. Delegates to diamond variant for diamonds (L240-L242). Otherwise scales the 4 aabb corners outward from midpoint by 2× (`pointScaleFromOrigin`) to form 4 large triangular "search cones" sharing the midpoint, then uses `triangleIncludesPoint` to classify `p` into UP/RIGHT/DOWN/LEFT (L244-L280). Comment (L228-L230) documents the cone algorithm. Coordinate-space: global points; `aabb` is the close-fitting (possibly rotated-aware) bounds.
- **`flipHeading(h)` → `Heading`** (L283-L287). Negates each non-zero component.

### packages/element/src/image.ts

Helpers for `ExcalidrawImageElement`: async image loading into a cache, SVG normalization, and type-narrowing collectors.

- **`loadHTMLImageElement(dataURL)` → `Promise<HTMLImageElement>`** (L21-L32). Wraps `new Image()` with onload/onerror; resolves the decoded image, rejects on error. Side effect: sets `image.src`.
- **`updateImageCache({ fileIds, files, imageCache })` → `Promise<{ imageCache, updatedFiles, erroredFiles }>`** (L36-L89). For each unique fileId with file data, stores an in-progress entry `{ image: <promise>, mimeType }` immediately (L67) so concurrent callers see initialization underway, then awaits and overwrites with the resolved `image` (L69-L71). `MIME_TYPES.binary` throws "Only images…" (L56-L58). Errors are caught and recorded in `erroredFiles` (cache still considered updated). Returns the (mutated) `imageCache`, `updatedFiles`, `erroredFiles`. NOTE in header (L34-L35): always overwrites cache even if populated — filter upstream to optimize. Invariant: `imageCache` is mutated in place.
- **`getInitializedImageElements(elements)` → `InitializedExcalidrawImageElement[]`** (L91-L96). Filters via `isInitializedImageElement`.
- **`isHTMLSVGElement(node)` → `node is SVGElement`** (L98-L102). Case-insensitive `nodeName === "svg"` test (XML/HTML case sensitivity note).
- **`normalizeSVG(SVGString)` → `string`** (L104-L153). Parses SVG via `DOMParser`; throws "Invalid SVG" on `parsererror` or missing `<svg>` (L107-L109). Ensures `xmlns` (L111-L113). Strips `%`/`auto` width/height to avoid scaling issues at different zooms (L115-L125). If width/height missing, defaults to 50 or extracts from `viewBox` via regex `/\d+ +\d+ +(\d+(?:\.\d+)?) +(\d+(?:\.\d+)?)/` (the 3rd/4th numbers) (L129-L144). Ensures a `viewBox` exists (L147-L149). Returns `svg.outerHTML`.

### packages/element/src/linearElementEditor.ts

The linear-element (line/arrow/elbow-arrow) interactive editing state machine: a `LinearElementEditor` value class holding per-edit-session state, plus a large set of static methods that mutate elements during point drag/add/delete, midpoint insertion, binding, and bounding-box computation. The single most geometry-heavy file in the cluster (~2500 lines). All point math is local↔global with element rotation about its bbox center.

Module-private:
- **`getNormalizedPoints({ points })` → `{ points: LocalPoint[]; offsetX; offsetY }`** (L97-L116). Shifts all points so `points[0]` becomes `[0,0]`; returns the removed offset. Invariant enforced across codebase: linear element first point is `[0,0]`.
- **type `PointMoveOtherUpdates`** (L118-L123): optional `startBinding`/`endBinding`/`moveMidPointsWithElement`/`suggestedBinding`.

Class `LinearElementEditor` (L125-L2111):
- **Fields** (L126-L162): `elementId` (branded), `selectedPointsIndices`, `initialState` (formerly `pointerDownState`: holds `prevSelectedPointsIndices`, `lastClickedPoint`, `origin`, `segmentMidpoint {value,index,added}`, `arrowStartIsInside`, `altFocusPoint`), `isDragging`, `lastUncommittedPoint`, `lastCommittedPoint`, `pointerOffset {x,y}`, `hoverPointIndex`, `segmentMidPointHoveredCoords`, `hoveredFocusPointBinding`, `draggedFocusPointBinding`, `elbowed`, `customLineAngle`, `isEditing`. `pointerDownState` is a deprecated `never` alias.
- **constructor(element, elementsMap, isEditing=false)** (L164-L205). Sets `elementId`; if `points[0]` is not `[0,0]` logs an error and self-heals via `mutateElement(... getNormalizeElementPointsAndCoords)` (L172-L179). Initializes all fields; `elbowed = isElbowArrow && element.elbowed`.

Runs `pointDraggingUpdates` → `movePoints` (L540-L570). Derives `suggestedBinding` only if start xor end selected; raises z-index (L572-L594). Resizes bound text via `handleBindTextResize` (L596-L600). For elbow arrows recomputes selected indices to `[0]` or `[last]` (L602-L612). Determines if the segment at `index` is horizontal; updates/creates a `FixedSegment` whose start/end pin one axis to the dragged coordinate and the other to the existing point (L2035-L2059); re-sorts fixed segments by index; mutates element; returns updated hovered-midpoint state pointing at the new segment center.
- **`deleteFixedSegment(element, scene, index)`** (L2100-L2110). Removes the fixed segment with matching `index`.

- **`pointDraggingUpdates(...)` → `{ positions: PointsPositionUpdates; updates?: PointMoveOtherUpdates }`** (L2123-L2501). The binding-aware point-move computation. When neither endpoint dragged but bound, recomputes bound endpoints via `updateBoundPoint` (L2213-L2266). When both/neither dragged equally, returns naive (L2268-L2272). The main path computes next `startBinding`/`endBinding` (mode null=unbind, focus point via `calculateFixedPointForNonElbowArrowBinding`), `suggestedBinding` (orbit mode or no COMPLEX_BINDINGS flag → outline-snap midpoint) (L2274-L2354). Builds a simulated `nextArrow`, handles the special case of an arrow dragged so both ends bind to the *same* element (`startIsDraggingOverEndElement`/`endIsDraggingOverStartElement`, gated by `COMPLEX_BINDINGS` flag and `bindMode`) (L2356-L2458), recomputes the non-dragged bound endpoint, and emits only the changed point indices in `positions` (L2460-L2500). The densest binding/geometry logic in the file.
- **`determineCustomLinearAngle(pivotPoint, draggedPoint)`** (L2503-L2507). `Math.atan2(dy, dx)` — the angle of the dragged segment for shift-lock baseline.

### packages/element/src/mutateElement.ts

Low-level element mutation primitives that bump version metadata for collaboration; does NOT trigger React re-render (use `scene.mutateElement` for that).

- **type `ElementUpdate<TElement>`** (L24-L27). `Omit<Partial<TElement>, "id"|"updated">`.
- **`mutateElement<TElement>(element, elementsMap, updates, options?)` → `TElement`** (L37-L144). In-place mutation. Elbow arrows: when updates are empty (normalization), or `points`/`fixedSegments` change, recomputes routing via `updateElbowArrowPoints` and forces `angle:0` (L53-L72). Non-elbow with `points`: derives width/height via `getSizeFromPoints` (L73-L75). Then per-key apply with change detection: skips equal primitives; for objects always updates *except* `groupIds`/`scale` special cases (L80-L97); for `points` does element-wise comparison (loops from the end via `while(--index)`, NOTE: index 0 not compared) and skips if unchanged (L98-L119). Sets `didChange`; if nothing changed returns element unmutated (L126-L128). Invalidates `ShapeCache` when width/height/fileId/points changed (L130-L137). Bumps `version` (`updates.version ?? +1`), `versionNonce` (`randomInteger()`), `updated` timestamp (L139-L141). Side effects: mutates `element`, mutates `ShapeCache`.
- **`newElementWith<TElement>(element, updates, force=false)` → `TElement`** (L146-L178). Immutable variant: returns a *new* object with updates applied; same change detection (objects always change unless null). Returns same reference if nothing changed and not forced. Always bumps version/versionNonce/updated on a real change.
- **`bumpVersion<T>(element, version?)`** (L185-L193). In-place: `version = (version ?? element.version)+1`, new `versionNonce`, new `updated`. Returns element.

### packages/element/src/newElement.ts

Element constructor factory functions producing fully-initialized `ExcalidrawElement` variants, plus text dimension/position adjustment helpers.

- **type `ElementConstructorOpts`** (L53-L76). `MarkOptional` over the generic element minus `id/type/isDeleted/updated`, with a long list of optional style/layout fields.
- **`_newElementBase<T>(type, opts)`** (internal, L78-L156). Builds the common element shape: applies `DEFAULT_ELEMENT_PROPS` defaults, `width/height=0`, `angle=0`, empty `groupIds`, `seed = randomInteger()`, `version=1`, `versionNonce=0`, `isDeleted:false`, fresh `updated`. Dev sanity check: logs an error if any of x/y/width/height exceeds ±1e6 (temporary guard, L103-L123). Returns a typed generic element.
- **`newElement(opts)` → `NonDeleted<ExcalidrawGenericElement>`** (L158-L163). Thin wrapper over `_newElementBase`.
- **`newEmbeddableElement(opts)`** (L165-L171). `_newElementBase("embeddable")`.
- **`newIframeElement(opts)`** (L173-L181). `_newElementBase("iframe")`.
- **`newFrameElement(opts)`** (L183-L198). Base + `type:"frame"` + `name` (default null), passed through `newElementWith` (to bump version).
- **`newMagicFrameElement(opts)`** (L200-L215). Same for `"magicframe"`.
- **`getTextElementPositionOffsets(opts, metrics)`** (internal, L217-L237). Computes x offset (center→w/2, right→w, else 0) and y offset (middle→h/2, else 0) from text/vertical alignment + measured metrics. Used to reposition text so its anchor stays fixed when its size changes.
- **`newTextElement(opts)` → `NonDeleted<ExcalidrawTextElement>`** (L239-L291). Resolves font family/size/lineHeight defaults, normalizes text, **measures** it via `measureText(getFontString(...))`, computes alignment offsets and adjusts `x/y` so the anchor is preserved, sets width/height to measured metrics, `originalText`, `autoResize` (default true). Wraps in `newElementWith`.
- **`getAdjustedDimensions(element, elementsMap, nextText)` → `{x,y,width,height}`** (internal, L293-L371). Recomputes text element bounds for new text. If not auto-resizing, keeps width. Center+middle non-contained auto-resize case repositions by the *difference* in measured metrics (L317-L334). Otherwise computes old vs new absolute coords (`getElementAbsoluteCoords`/`getResizedElementAbsoluteCoords`), halves the corner deltas, and applies rotation-aware repositioning via `adjustXYWithRotation` so the visible anchor edge stays put under rotation (L335-L362). Guards against non-finite x/y.
- **`adjustXYWithRotation(sides, x, y, angle, deltaX1, deltaY1, deltaX2, deltaY2)` → `[number,number]`** (internal, L373-L418). Adjusts x/y for a resize given which sides (n/e/s/w) moved, accounting for element rotation: applies `cos`/`sin` of `angle` to distribute the corner deltas. Key rotation math for resize parity — when both opposite sides move it's a simple sum; single-side moves apply the `(1±cos)`/`±sin` distribution.
- **`refreshTextDimensions(textElement, container, elementsMap, text=textElement.text)`** (L420-L440). Returns null if deleted; wraps text (`wrapText`) when contained or not auto-resizing (width from container max width or element width), then returns `{ text, ...getAdjustedDimensions(...) }`.
- **`newFreeDrawElement(opts)`** (L442-L456). Base + `points`/`pressures` (default `[]`) + `simulatePressure`.
- **`newLinearElement(opts)` → `NonDeleted<ExcalidrawLinearElement>`** (L458-L485). Base + empty `points`, null bindings/arrowheads; if it's a line element, adds `polygon` (default false) and returns the narrowed line type.
- **`newArrowElement<T extends boolean>(opts)`** (L487-L525). Conditional return type: `elbowed` true → `ExcalidrawElbowArrowElement` with `elbowed:true`, `fixedSegments:[]`, `startIsSpecial/endIsSpecial:false`; else a plain arrow with `elbowed:false`. Both default null bindings + optional arrowheads.
- **`newImageElement(opts)` → `NonDeleted<ExcalidrawImageElement>`** (L527-L546). Base with `strokeColor:"transparent"` (comment: future per-SVG stroke), `status` (default "pending"), `fileId` (default null), `scale` (default `[1,1]`), `crop` (default null).

Notable cross-file geometry/coordinate-space details for parity:
- Linear element points are stored in **local space with `points[0] === [0,0]`** as a hard invariant; global coords are derived by `+element.x/y` then rotation about the **AABB center** `((x1+x2)/2,(y1+y2)/2)` by `element.angle`. Moving point 0 is implemented by offsetting all other points and element x/y in the opposite direction (linearElementEditor `movePoints`/`_updatePoints`).
- Heading geometry uses a **y-down** convention (`HEADING_DOWN = [0,1]`) and resolves headings via search cones (2× scaled AABB corners + `triangleIncludesPoint`) for non-diamonds and per-vertex cross-product sectors (with 0.95 shrink) for diamonds.
- AABB of linear elements is derived from the **rendered RoughJS curve path ops**, not raw points (`getElementAbsoluteCoords` via `ShapeCache`).
- Hit-test thresholds are screen-space: `POINT_HANDLE_SIZE(=10)` compared against `pointDistance * zoom.value`.
- Version bookkeeping uses **djb2** hashing (`hashElementsVersion`/`hashString`) coerced to unsigned 32-bit.
