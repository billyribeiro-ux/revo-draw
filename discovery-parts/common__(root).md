## Cluster: common__(root)

This cluster contains two files at the root of the `@excalidraw/common` package: a development-only performance/diff-debugging utility (`debug.ts`) and a TypeScript ambient-reference declaration file (`global.d.ts`).

### packages/common/debug.ts

Purpose: A development/instrumentation helper that records per-named timing samples (one-shot, per-frame-averaged, and `console.time`-wrapped), auto-starts/stops a 1-second logging interval driven by `requestAnimationFrame` frame counting, and provides a deep-equality object-diff logger ‚Äî none of which affects production rendering output.

- `lessPrecise(num: number, precision = 5) => number` ‚Äî L1-L2. Rounds a number to `precision` significant figures via `num.toPrecision(precision)` then re-parses with `parseFloat` to drop trailing zeros and return a plain `number`. Module-private; used everywhere a logged time is printed. Pure, no side effects. Invariant: output keeps at most `precision` significant digits (default 5).
- `getAvgFrameTime(times: number[]) => number` ‚Äî L4-L5. Computes the arithmetic mean of `times` (`reduce((a,b)=>a+b)/length`) and passes it through `lessPrecise`. Module-private. Note: uses `reduce` without an initial value, so it throws on an empty array (caller must guarantee non-empty; callers gate on `times.length`). Pure.

Class `Debug` (L7-L181) ‚Äî a static-only namespace class (never instantiated). Static fields:
  - `DEBUG_LOG_TIMES = true` (L8) ‚Äî public master on/off switch for emitting logs in `debugLogger`.
  - `TIMES_AGGR: Record<string, { t: number; times: number[] }>` (L10-L11) ‚Äî private per-name aggregation buckets for `logTime`; `t` is the last `performance.now()` start timestamp, `times` accumulates per-call deltas.
  - `TIMES_AVG: Record<string, { t: number; times: number[]; avg: number | null }>` (L12-L15) ‚Äî private per-name buckets for `logTimeAverage`; additionally carries a running `avg`.
  - `LAST_DEBUG_LOG_CALL = 0` (L16) ‚Äî private; epoch-ms (`Date.now()`) of the most recent `logTime`/`logTimeAverage` call, used as the idle-stop trigger.
  - `DEBUG_LOG_INTERVAL_ID: null | number = null` (L17) ‚Äî private; `setInterval` handle; `null` doubles as the "recording stopped" sentinel.
  - `LAST_FRAME_TIMESTAMP = 0`, `FRAME_COUNT = 0`, `ANIMATION_FRAME_ID: null | number = null` (L19-L21) ‚Äî private; rAF frame-counting state used to compute "ms/frame" and "% of frame budget".

Methods (all static, arrow-function properties unless noted):
  - `scheduleAnimationFrame = () => void` ‚Äî L23-L36. Self-rescheduling `requestAnimationFrame` loop. Only schedules while `DEBUG_LOG_INTERVAL_ID !== null`. On each frame, if the `timestamp` differs from `LAST_FRAME_TIMESTAMP` it records the new timestamp and increments `FRAME_COUNT`, then re-arms itself. Side effect: stores the rAF handle in `ANIMATION_FRAME_ID`. Invariant: counts at most one frame per distinct rAF timestamp (de-dupes coalesced callbacks).
  - `setupInterval = () => void` ‚Äî L38-L45. Lazily starts perf recording: if no interval is running, logs `"(starting perf recording)"` (green), starts a 1000 ms `window.setInterval(Debug.debugLogger, 1000)`, and kicks off `scheduleAnimationFrame`. Always updates `LAST_DEBUG_LOG_CALL = Date.now()` to keep the recording alive. Side effects: timer + rAF registration, console output.
  - `debugLogger = () => void` ‚Äî L47-L100. The interval callback. When `DEBUG_LOG_TIMES`: (1) for each `TIMES_AGGR` entry with samples, logs name + total time (`lessPrecise(sum)`) + the sorted (ascending) per-call times, then clears that bucket's `times` (L49-L58); (2) for each `TIMES_AVG` entry with samples, computes `avgFrameTime = lessPrecise(totalTime / FRAME_COUNT)` and logs "N calls - X ms/frame across F frames (P% of frame budget)" where `P = (avgFrameTime / 16.67) * 100` rounded to 1 sig-fig, then resets `times` and folds the new `avgFrameTime` into the running `avg` via `getAvgFrameTime([avg, avgFrameTime])` (L59-L80). Resets `FRAME_COUNT = 0` (L82). Stop condition (L84-L99): if more than 600 ms have elapsed since `LAST_DEBUG_LOG_CALL` and an interval is active, logs `"(stopping perf recording)"` (red), clears the interval, cancels the rAF (`ANIMATION_FRAME_ID!`), and zeroes all state (`FRAME_COUNT`, `LAST_FRAME_TIMESTAMP`, `DEBUG_LOG_INTERVAL_ID = null`, empties `TIMES_AGGR`/`TIMES_AVG`). Notable math/perf: `16.67` is the hardcoded ms-per-frame budget for ~60 fps; "% of frame budget" is per-frame avg time divided by that budget. Dividing `totalTime / FRAME_COUNT` divides by frames, not by call count, so it expresses cost amortized across rendered frames.
  - `logTime = (time?: number, name = "default") => void` ‚Äî L102-L113. Public one-shot timer. Calls `setupInterval()` (auto-starts recording / refreshes liveness). Lazily creates the `TIMES_AGGR[name]` bucket. If a prior start `t` exists, pushes either the explicit `time` argument or the measured delta `performance.now() - t`. Always updates `t` to the current `performance.now()`. Side effect: mutates `TIMES_AGGR`. Invariant: the first call only sets `t` (no sample pushed) since `t` is initially `0`/falsy.
  - `logTimeAverage = (time?: number, name = "default") => void` ‚Äî L114-L125. Same as `logTime` but writes into `TIMES_AVG[name]` (the per-frame-averaged bucket). Same `setupInterval`/first-call-seeds-`t` behavior.
  - `logWrapper = (type: "logTime" | "logTimeAverage") => <T extends any[], R>(fn: (...args: T) => R, name = "default") => (...args: T) => R` ‚Äî L127-L136. Private higher-order factory. Returns a function that, given any function `fn`, returns a wrapped version measuring `performance.now()` around the call and forwarding the elapsed time to `Debug[type](elapsed, name)`, then returns `fn`'s original return value. Generic over arg-tuple `T` and return `R`; transparent to callers.
  - `logTimeWrap = Debug.logWrapper("logTime")` (L138) ‚Äî public; wraps a function to record one-shot timings.
  - `logTimeAverageWrap = Debug.logWrapper("logTimeAverage")` (L139) ‚Äî public; wraps a function to record per-frame-averaged timings.
  - `perfWrap = <T extends any[], R>(fn, name = "default") => (...args: T) => R` ‚Äî L141-L153. Public. Wraps `fn` in `console.time(name)` / `console.timeEnd(name)`, returning `fn`'s result. Independent of the interval system; emits browser-native timing marks. Has inline eslint-disable for `no-console`.
  - `CHANGED_CACHE: Record<string, Record<string, unknown>>` (L155) ‚Äî private; stores the previous object snapshot per name for `logChanged`.
  - `logChanged(name: string, obj: Record<string, unknown>): void` ‚Äî L157-L180. Public regular method (not arrow). Diffs `obj` against the previously cached object for `name`. Stores the new `obj` as the cache (L160), returns early if there was no prior snapshot. Builds the union of prev+next keys (L166), and for each key uses `deepEqual` to detect changes, collecting `{ prev, next }` per changed key. If any changed, logs `"[name] changed:"` with the diff map. Side effect: console output + cache mutation. Invariant: stores `obj` by reference (not cloned), so later in-place mutation of `obj` between calls would defeat the diff.

- `deepEqual(a: unknown, b: unknown): boolean` ‚Äî L183-L220. Module-private recursive structural equality. Order: `Object.is(a,b)` short-circuit (handles primitives, identity, NaN, ¬±0 distinction) (L184-L186); returns false if either is `null` or non-object (L188-L195); returns false if array-ness differs (L197-L199); compares own-enumerable key counts (L201-L205) then recurses on each of `a`'s keys (L208-L218). Notable: only iterates `keysA` in the recursion, relying on equal key counts to imply equal key sets ‚Äî a key present in `b` but absent in `a` (with matching count) is not separately checked, which is sound only because the count check plus same-key recursion catches mismatches in practice. Does not handle Maps/Sets/Dates/RegExp specially (compares them as plain objects, i.e. by own enumerable keys). Pure, no side effects.

Performance/coordinate notes: this file is purely an instrumentation utility ‚Äî no canvas, geometry, or coordinate-space logic. The only "magic" numbers are `16.67` (60 fps frame budget in ms), the `600` ms idle-stop threshold, the `1000` ms logging interval, and significant-figure precisions (`5` default, `1` for the percentage). It depends on browser globals `window.setInterval`/`clearInterval`, `requestAnimationFrame`/`cancelAnimationFrame`, `performance.now()`, `Date.now()`, and `console.*`, so it is browser-only (not safe in a pure Node test env without those globals).

### packages/common/global.d.ts

Purpose: A 3-line ambient TypeScript declaration / side-effect type-import file that wires Vite client types and the published Excalidraw global + CSS module typings into the package's type scope. Types-only ‚Äî contains no runtime code and no functions, classes, or constants.

- L1: `/// <reference types="vite/client" />` ‚Äî triple-slash directive pulling in Vite's client ambient types (e.g. `import.meta.env`, asset import module declarations).
- L2: `import "@excalidraw/excalidraw/global";` ‚Äî side-effect import of the main package's global ambient declarations.
- L3: `import "@excalidraw/excalidraw/css";` ‚Äî side-effect import providing typings for CSS imports.

No exported or internal symbols to document; nothing to compare for runtime parity.
