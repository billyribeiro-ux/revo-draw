## Audit: color-picker

Surgical side-by-side re-verification of the just-rebuilt picker box in `StylePanel.svelte`
(`{#snippet pickerBox}`, lines 150–198, styles 431–499) and `palette.ts`, against Excalidraw's
`Picker.tsx` / `PickerColorList.tsx` / `ShadeList.tsx` / `ColorInput.tsx` and `ColorPicker.scss`.

Scope: base grid, shade row, hex input geometry, swatch sizes/colors, active ring. The hex values,
5-column grid (`repeat(5, 1.875rem)`, gap `0.25rem`), padding `0.5rem`, grid-swatch size
`1.875rem` + radius `0.5rem` + `inset 0 0 0 1px #d9d9d9` outline, headings "Colors"/"Shades", and
`#`-prefixed hex with `0.4px` letter-spacing all match Excalidraw exactly and are NOT flagged.

### By-design divergences (do NOT fix)

- **No eyedropper trigger.** Excalidraw's `ColorInput.tsx:96-127` renders an eyedropper button + `var(--default-border-color)` divider beside the hex input (with `I`/`Alt` shortcut). This is a Canvas/photo-tool affordance irrelevant to a semantic-layout export tool; omitting it is intentional.
- **No "most used custom colors" row.** `Picker.tsx:171-183` + `CustomColorList` shows recently-used custom colors harvested from canvas elements. Our tool's color model and Markdown export don't track custom-color frequency; skip.
- **No canvas-background picker variant.** `ColorPicker.scss:278-296` (`--canvas`) is for the infinite-canvas background; not part of element styling here.
- **No connector triangle / popover positioning chrome.** `ColorPicker.scss:226-256` (`.color-picker-triangle`) and the jotai-atom-driven popover container are Excalidraw's Radix-popover styling; our `.grid-pop` anchors to the trigger's right edge by design to stay inside the 240px panel.
- **Accent hue differs (oklch purple vs `#6965db`).** Our `--accent` is the product's own brand token, not Excalidraw's `--color-primary`. Only the grid active-ring *darkness relationship* is flagged above (use a `darkest` step), not the base hue.
- **Theme filter / dark-mode hue-rotate pipeline.** `colors.ts:83-110` (`applyDarkModeFilter`, `--theme-filter`) inverts+hue-rotates swatches in dark mode. We don't replicate Excalidraw's CSS-filter dark mode; intentional.
