## Audit: style-panel

Side-by-side parity audit of the style/property panel. OUR implementation is `src/lib/ui/StylePanel.svelte` (Svelte 5 + Canvas2D). Excalidraw counterpart is the set of property actions in `actions/actionProperties.tsx` rendered via `RadioSelection`/`Range`/`ColorPicker`, styled by `components/Actions.scss`, `css/styles.scss` (`.selected-shape-actions`, `.buttonList`), `components/Range.scss`, and the `outlineButtonStyles` mixin in `css/variables.module.scss`.

### By-design divergences (do NOT fix)

- **Semantic style model, not Excalidraw element model.** OUR panel exposes `stroke`, `fill`, `strokeWidth` (`thin/bold/extra`), `strokeStyle` (`solid/dashed/dotted`), `opacity`. Excalidraw additionally has fill style (hachure/cross-hatch/solid/zigzag), sloppiness/roughness, edge roundness, arrowheads, arrow type, font family/size/text-align/vertical-align. Those map to rough.js rendering and Excalidraw's rectangle/ellipse/arrow/text primitives — out of scope for a clean Canvas2D semantic-layout tool. Do not add them.
- **No `roughness`/`fillStyle`/`seed`/`randomInteger` (`actionProperties.tsx:465-657`).** OUR fills are flat; hand-drawn hachure/zigzag is explicitly a rough.js feature we don't reproduce.
- **Panel is a floating canvas Island (top-left), not Excalidraw's left sidebar `.selected-shape-actions` column.** OUR row-based horizontal layout (label-left, control-right) is an intentional compact-panel design choice; only the label *typography* and active-button *coloring* above are flagged as matchable, not the column-vs-row layout itself.
- **Opacity precision (min=0/max=1/step=0.05, 21 stops) vs Excalidraw min=0/max=100/step=10 (11 stops).** OUR finer granularity is a deliberate product improvement, not a defect — keep it; only the missing value readout/track-fill chrome is matchable.
- **No mobile/compact `stylesPanelMode` branching** (`actionProperties.tsx:158-165`, `deriveStylesPanelMode`) and no `compact-shape-actions`/`mobile-shape-actions` (`Actions.scss:95-208`). Single-user desktop Tauri app — mobile/compact modes are out of scope.
- **No font/text actions, arrow actions, polygon toggle, cardinality arrowheads.** Tied to Excalidraw element types and collab features; out of scope.
