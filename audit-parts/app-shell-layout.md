## Audit: app-shell-layout

Axis focus: overlay/island layout, top 3-column zone, z-index layering, pointer-events model, responsive breakpoints.

Scope of comparison: our web shell in `src/routes/+page.svelte` (the `{#if isWeb}` branch, lines 498-613, plus its `<style>` block lines 673-984) against Excalidraw's `LayerUI.tsx` `renderFixedSideContainer()` (lines 287-434) and `css/styles.scss` / `LayerUI.scss` / `FixedSideContainer.scss` / `theme.scss` / `editorInterface.ts`.

Token verification: the web build loads `src/lib/ui/excalidraw-web.css` (`.x-web` scope) which already pins `--shadow-island`, `--radius-lg` (0.5rem), `--space-4` (1rem = `--editor-container-padding`) to Excalidraw's exact values. Those are NOT findings — they match. The findings below are the structural / behavioral / responsive gaps that the token remap does not cover.

### By-design divergences (do NOT fix)

- **No `--ui-pointerEvents` CSS variable / zen-mode / view-mode toggling.** Excalidraw toggles `--ui-pointerEvents` between `all`/`none` for zen and view modes (`styles.scss:288,321,348,361`). We hardcode `pointer-events: auto` on islands. We have no zen/view mode — out of scope.
- **No `FixedSideContainer` safe-area-inset padding.** Excalidraw pads the top container with `env(safe-area-inset-*)` (`styles.scss:141-146`) for notched mobile/PWA. This is a single-user desktop/web app, not a mobile PWA.
- **No `MobileMenu` / phone form-factor component.** Excalidraw swaps the entire chrome for `MobileMenu` when `formFactor === "phone"` (`LayerUI.tsx:571-588`). We instead reflow panels to bottom sheets. Different product, mobile collab out of scope per guardrail.
- **No collaborator `UserList` / laser-pointer island / sidebar-dock width reservation.** `LayerUI.tsx:369-388,404-409,591-599` (`width: calc(100% - var(--right-sidebar-width))`). No collab in this product.
- **No `Stats`, `HintViewer`, `PenModeButton`, `Section heading` a11y wrappers in the top zone.** These are Excalidraw-specific affordances tied to its element model and pen/stats features; our semantic-layout tool has its own StylePanel/RightPanel.
- **Panel-dock as floating island vs. Excalidraw docked sidebar.** Our right/left panels float over the canvas (`+page.svelte:835-848`) rather than reserving a docked column (`LayerUI.tsx:593-599`). Intentional: full-bleed canvas is the chosen model and the canvas owns the viewport on blank load (comment at `+page.svelte:41`).
- **Tauri desktop shell branch** (`+page.svelte:614-652`, TitleBar + tool rail + StatusBar) is explicitly out of the web-parity scope per the inline comment at lines 615-616.
